# End-to-End Tests

このディレクトリには、Playwrightを使用したエンドツーエンドテストが含まれています。

## テストファイル

### config-auth.spec.ts
設定ページの認証機能をテストします：
- ログインフォームの表示
- 有効/無効なパスワードでの認証
- ログアウト機能

### config-edit.spec.ts
設定の編集機能とバリデーション機能をテストします：
- Edit ボタンの表示
- モーダルの開閉
- 既存値の事前入力
- バリデーションルール機能
- 新しい設定の追加
- 既存設定の更新

## テストの実行

### 基本的な実行
```bash
npm run test:e2e
```

### UIモードで実行（デバッグに便利）
```bash
npm run test:e2e:ui
```

### ヘッドレスモードではなくブラウザを表示して実行
```bash
npm run test:e2e:headed
```

### 特定のテストファイルのみ実行
```bash
npx playwright test config-edit.spec.ts
```

### 特定のブラウザでのみ実行
```bash
npx playwright test --project=chromium
```

## 前提条件

1. 開発サーバーが起動していること（`npm run dev`）
2. データベースに適切な設定データが存在すること
3. 認証パスワードが設定されていること

## テスト設定

- **Base URL**: `http://localhost:8787`
- **設定パスワード**: `testPassword123`（テスト内で定義）
- **タイムアウト**: 10秒（認証待機）

## 注意事項

- テストは実際のブラウザ環境で実行されるため、開発サーバーが正常に動作している必要があります
- テストは順次実行され、各テストは独立しています
- 認証状態はテスト間でリセットされます
