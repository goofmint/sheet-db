# [2.2.11] DELETE /api/v1/configs/:keyを利用した/config での設定削除

## ステータス

## 概要
/config ページにおいて、DELETE /api/v1/configs/:key APIを使用してクライアントサイドから既存の設定項目を削除できるようにする。削除操作には適切な確認ダイアログを実装し、誤操作を防ぐ。

## 設計

### 1. UI/UXデザイン

#### 削除ボタンの配置
- **場所**: 各設定行のActionsカラムに「Delete」ボタンを追加
- **スタイル**: 
  - クラス: `btn btn-sm btn-delete`
  - 色: 赤系統（danger color）で視覚的に注意を促す
  - アイコン: ゴミ箱アイコン（オプション）
- **位置**: Editボタンの右側に配置

#### 確認ダイアログ
- **モーダルダイアログ**:
  - タイトル: "Delete Configuration"
  - メッセージ: "Are you sure you want to delete '{key}'? This action cannot be undone."
  - ボタン:
    - Cancel (デフォルトフォーカス)
    - Delete (赤色、danger style)
- **キーボード操作**:
  - Escapeキーで閉じる（IME確定中は無効）
  - Enterキーは無効化（誤操作防止）

#### JavaScript実装
```javascript
// 削除対象の設定を保持
let deletingConfig = null;

// 削除確認モーダルを表示
function showDeleteConfirmModal(config) {
  deletingConfig = config;
  
  // キー名を表示
  document.getElementById('delete-config-key').textContent = config.key;
  
  // モーダルを表示
  document.getElementById('delete-confirm-modal').style.display = 'block';
  
  // Cancelボタンにフォーカス（安全のため）
  document.getElementById('delete-cancel-btn').focus();
}

// 削除確認モーダルを非表示
function hideDeleteConfirmModal() {
  document.getElementById('delete-confirm-modal').style.display = 'none';
  deletingConfig = null;
}

// 設定を削除
async function deleteConfig() {
  if (!deletingConfig) return;
  
  const deleteBtn = document.getElementById('delete-confirm-btn');
  deleteBtn.disabled = true;
  deleteBtn.textContent = 'Deleting...';
  
  try {
    const response = await fetch(`/api/v1/configs/${encodeURIComponent(deletingConfig.key)}`, {
      method: 'DELETE',
      headers: {
        'Authorization': `Bearer ${configPassword}`,
        'X-CSRF-Token': getCSRFToken() // CSRF対策
      }
    });
    
    if (!response.ok) {
      const error = await response.json();
      throw new Error(error.error?.message || `HTTP ${response.status}`);
    }
    
    // 成功
    const deletedKey = deletingConfig.key; // preserve before reset
    hideDeleteConfirmModal();
    showSuccessMessage(`Configuration '${deletedKey}' deleted successfully.`);
    
    // 設定リストを再読み込み
    await loadConfigsWithPassword(configPassword);
    
  } catch (error) {
    console.error('Failed to delete config:', error);
    showErrorMessage(`Failed to delete configuration: ${error.message}`);
  } finally {
    deleteBtn.disabled = false;
    deleteBtn.textContent = 'Delete';
  }
}

// テーブル更新時にDeleteボタンを追加
function updateConfigTable(configs) {
```

### 3. セキュリティ考慮事項

#### CSRF保護
- 削除リクエストには必ずCSRFトークンを含める
- トークンはセッションストレージから取得
- ヘッダー名: `X-CSRF-Token`

#### 認証・認可
- Bearer認証によるアクセス制御
- config_passwordを使用した認証
- 削除権限の確認（サーバー側）

#### 二重削除防止
- 削除処理中はボタンを無効化
- ローディング状態の表示
- エラー時の適切なハンドリング

### 4. エラーハンドリング

#### エラーケース
1. **認証エラー (401/403)**
   - メッセージ: "Authentication failed. Please login again."
   - 対応: ログイン画面へリダイレクト

2. **設定が見つからない (404)**
   - メッセージ: "Configuration not found. It may have been already deleted."
   - 対応: リストを再読み込み

3. **競合エラー (409)**
   - メッセージ: "Configuration was modified. Please refresh and try again."
   - 対応: リストを再読み込み

4. **サーバーエラー (500)**
   - メッセージ: "Server error occurred. Please try again later."
   - 対応: エラーメッセージ表示
