## 概要
Auth0から発行されるJWTトークンの検証機能を実装する。これにより、API認証の安全性を向上させ、トークンの改ざんや期限切れを検出できるようにする。

## 設計

### 1. JWTトークン検証の要件

#### 検証項目
- **署名検証**: Auth0の公開鍵を使用してトークンの署名を検証
- **発行者（iss）**: Auth0のドメインと一致することを確認
- **対象者（aud）**: アプリケーションのClient IDと一致することを確認
- **有効期限（exp）**: トークンが有効期限内であることを確認
- **発行時刻（iat）**: トークンが未来の時刻で発行されていないことを確認
- **有効開始時刻（nbf）**: トークンがまだ有効になっていないことを確認
- **トークンID（jti）**: リプレイ攻撃防止のための一意性確認（オプション）

#### 公開鍵の取得
- Auth0のJWKS（JSON Web Key Set）エンドポイントから公開鍵を取得
- エンドポイント: `https://{AUTH0_DOMAIN}/.well-known/jwks.json`
- 公開鍵はキャッシュして、パフォーマンスを向上

### 2. 実装詳細

#### JWTサービスの作成
```typescript
// src/services/jwt.ts
export interface JWTPayload {
  sub: string;           // ユーザーID
  iss: string;           // 発行者
  aud: string | string[]; // 対象者
  exp: number;           // 有効期限
  iat: number;           // 発行時刻
  nbf?: number;          // 有効開始時刻（not-before）
  jti?: string;          // トークンID
  email?: string;        // メールアドレス
  email_verified?: boolean;
  roles?: string[];      // ユーザーロール
}

export interface JWTVerifyOptions {
  issuer: string;
  audience: string;
  algorithms?: string[];
  clockTolerance?: number; // 時刻のずれ許容範囲（秒）
  ignoreNotBefore?: boolean; // nbfチェックを無視するか（デフォルト: false）
}

// シングルトンインスタンスを保持
let jwtServiceInstance: JWTService | null = null;

export class JWTService {
  private jwksClient: JwksClient;
  private publicKeyCache: Map<string, string>;
  
  constructor(jwksUri: string) {
    this.jwksClient = createJwksClient({
      jwksUri,
      cache: true,
      cacheMaxAge: 86400000, // 24時間
      rateLimit: true,
      jwksRequestsPerMinute: 10
    });
    this.publicKeyCache = new Map();
  }
  
  // シングルトンインスタンスを取得
  static getInstance(jwksUri: string): JWTService {
    if (!jwtServiceInstance) {
      jwtServiceInstance = new JWTService(jwksUri);
    }
    return jwtServiceInstance;
  }
  
  async verifyToken(token: string, options: JWTVerifyOptions): Promise<JWTPayload> {
    // 1. トークンのデコード（検証なし）
    const decoded = jwt.decode(token, { complete: true }) as
      | { header: { kid: string }; payload: any }
      | null;
    
    if (!decoded || !decoded.header?.kid) {
      throw new Error('Invalid token header');
    }
    
    // 2. 公開鍵の取得
    const publicKey = await this.getPublicKey(decoded.header.kid);
    
    // 3. トークンの検証
    const payload = jwt.verify(token, publicKey, {
      issuer: options.issuer,
      audience: options.audience,
      algorithms: options.algorithms || ['RS256'],
      clockTolerance: options.clockTolerance || 5,
      ignoreNotBefore: options.ignoreNotBefore || false
    });
    
    // 4. 追加の時刻検証（iatが未来でないことを確認）
    const now = Math.floor(Date.now() / 1000);
    if (payload.iat && payload.iat > now + (options.clockTolerance || 5)) {
      throw new Error('Token issued in the future');
    }
    
    return payload as JWTPayload;
  }
  
  private async getPublicKey(kid: string): Promise<string> {
    // キャッシュから取得
    if (this.publicKeyCache.has(kid)) {
      return this.publicKeyCache.get(kid)!;
    }
    
    // JWKSから取得
    const key = await this.jwksClient.getSigningKey(kid);
    const publicKey = key.getPublicKey();
    
    // キャッシュに保存
    this.publicKeyCache.set(kid, publicKey);
    
    return publicKey;
  }
}
```

#### 認証ミドルウェアへの統合
```typescript
// src/middleware/auth.ts の更新
export function auth(options: AuthOptions = {}): MiddlewareHandler {
  return async (c, next) => {
    // 既存のセッション認証に加えて、JWTトークン検証を追加
    
    // Bearerトークンのチェック
    const authHeader = c.req.header('Authorization');
    if (authHeader?.startsWith('Bearer ')) {
      const token = authHeader.substring(7);
      
      try {
        const jwtService = JWTService.getInstance(
          `https://${c.env.AUTH0_DOMAIN}/.well-known/jwks.json`
        );
        
        const payload = await jwtService.verifyToken(token, {
          issuer: `https://${c.env.AUTH0_DOMAIN}/`,
          audience: c.env.AUTH0_CLIENT_ID
        });
        
        // 認証コンテキストの設定
        authContext.isAuthenticated = true;
        authContext.userId = payload.sub;
        authContext.roles = payload.roles || [];
        
        return await next();
      } catch (error) {
        // JWTが無効な場合は、セッション認証にフォールバック
        console.error('JWT verification failed:', error);
      }
    }
    
    // 既存のセッション認証ロジック...
  };
}
```

### 3. エラーハンドリング

#### エラーケース
1. **署名検証エラー**
   - 原因: トークンが改ざんされている
   - 対応: 401 Unauthorized

2. **有効期限切れ**
   - 原因: トークンの有効期限が過ぎている
   - 対応: 401 Unauthorized with "Token expired"

3. **発行者不一致**
   - 原因: 異なるAuth0テナントのトークン
   - 対応: 401 Unauthorized with "Invalid issuer"

4. **対象者不一致**
   - 原因: 異なるアプリケーションのトークン
   - 対応: 401 Unauthorized with "Invalid audience"

5. **公開鍵取得エラー**
   - 原因: ネットワークエラーまたはJWKSエンドポイントの問題
   - 対応: 503 Service Unavailable

### 4. パフォーマンス最適化

#### キャッシュ戦略
- **公開鍵キャッシュ**: 24時間キャッシュ
- **JWKSレスポンスキャッシュ**: jwks-rsaライブラリの内蔵キャッシュを使用
- **検証結果キャッシュ**: 同一トークンの再検証を避ける（オプション）

#### レート制限
- JWKSエンドポイントへのリクエストは1分間に10回まで
- トークン検証自体にレート制限は設けない

### 5. セキュリティ考慮事項

#### トークンの取り扱い
- トークンをログに出力しない
- エラーメッセージにトークンの内容を含めない
- HTTPSでの通信を必須とする

#### 時刻の同期
- サーバーの時刻がずれている場合を考慮して、5秒の許容範囲を設定
- NTPによる時刻同期を推奨

### 6. テスト戦略

#### 単体テスト
- 正常なトークンの検証
- 期限切れトークンの拒否
- 改ざんされたトークンの検出
- 不正な発行者/対象者の検出

#### 統合テスト
- Auth0の実際のトークンを使用した検証
- 公開鍵の取得とキャッシュ
- エラーハンドリングの確認
