## 概要
Auth0ログインへのリダイレクトを行うAPIエンドポイント `src/api/auth/login/get.ts` を作成します。このエンドポイントは、ユーザーをAuth0の認証画面へリダイレクトし、認証後のコールバックを処理するための準備を行います。

## 実装内容

### 1. ログインAPIエンドポイント (`src/api/auth/login/get.ts`)

#### 主要機能
1. **Auth0認証URLの生成**
   - Auth0Serviceを使用してauthorization URLを生成
   - stateパラメータの生成（CSRF対策）
   - リダイレクトURIの動的生成

2. **セキュリティ対策**
   - CSRF攻撃対策のためのstateパラメータ
   - リダイレクトURIの検証（allowedRedirectBases配列から選択）
   - セッション管理（state情報の一時保存）

3. **エラーハンドリング**
   - Auth0設定が不完全な場合の処理
   - 無効なリダイレクトURIの処理
   - ネットワークエラーの処理

#### API設計

**エンドポイント**: `GET /api/auth/login`

**クエリパラメータ**: なし

**レスポンス**:
- **成功**: HTTP 302 Redirect to Auth0
- **エラー**: HTTP 400/500 with JSON error response

**ログイン完了後の動作**:
Auth0からコールバックされ、認証処理が成功した場合は、認証情報をJSONで返却します。

#### 実装例
```typescript
import { Hono } from 'hono';
import { Auth0Service } from '../../../services/auth0';
import { ConfigService } from '../../../services/config';
import { Env } from '../../../types/env';

const app = new Hono<{ Bindings: Env }>();

app.get('/', async (c) => {
  try {
    // Get current host and validate redirect URI
    const currentHost = new URL(c.req.url).origin;
    const allowedRedirectBases = JSON.parse(
      ConfigService.getString('allowedRedirectBases', '[]')
    ) as string[];
    
    const allowedBase = allowedRedirectBases.find(base => base === currentHost);
    if (!allowedBase) {
      return c.json({ error: 'Unauthorized redirect base URL' }, 400);
    }
    
    const redirectUri = `${allowedBase}/api/auth/callback`;
    
    // Generate state parameter for CSRF protection
    const state = crypto.randomUUID();
    
    // Store state in session/cache for validation
    // TODO: Implement state storage mechanism
    
    // Get Auth0 authorization URL
    const auth0Service = new Auth0Service(c.env);
    const authUrl = await auth0Service.getAuthorizationUrl(state, redirectUri);
    
    // Note: No post-login redirect URL needed
    // Auth0 callback will return JSON with authentication info
    
    return c.redirect(authUrl);
    
  } catch (error) {
    console.error('Login error:', error);
    return c.json({
      error: 'Authentication service unavailable',
      message: error instanceof Error ? error.message : 'Unknown error'
    }, 500);
  }
});

export default app;
```

### 3. セキュリティ実装

#### CSRF対策
- `crypto.randomUUID()` を使用したstate生成
- stateパラメータをセッションまたはキャッシュに一時保存
- コールバック時のstate検証

#### リダイレクトURI検証
- `allowedRedirectBases` 配列から現在のホストを検証
- 許可されていないホストからのリクエストを拒否
- 動的なリダイレクトURI生成

#### エラー情報の適切な処理
- 内部エラーの詳細を外部に漏らさない
- ログに十分な情報を記録
- ユーザーフレンドリーなエラーメッセージ
