## 概要
設定管理UI（`GET /config`）に設定値の編集・保存機能を追加する。管理者が各設定項目を動的に変更し、リアルタイムでアプリケーション設定を更新できる機能を実装する。

## 機能要件

### 設定値編集フォーム
- **目的**: 設定一覧画面で各設定項目を編集可能にする
- **機能**:
  - 設定値のインライン編集
  - データ型に応じた入力フィールド（テキスト、チェックボックス、セレクト）
  - リアルタイム保存またはバッチ保存

### 設定保存API
- **エンドポイント**: `POST /api/v1/config/update`
- **目的**: 設定値の更新をAPIで処理
- **機能**:
  - 個別設定項目の更新
  - 複数設定項目の一括更新
  - 設定値のバリデーション

## 実装仕様

### POST /api/v1/config/update - 設定更新API

#### リクエスト形式
```typescript
interface ConfigUpdateRequest {
  updates: {
    key: string;
    value: string;
    type?: 'string' | 'boolean' | 'number';
  }[];
}
```

#### レスポンス形式
```typescript
interface ConfigUpdateResponse {
  success: boolean;
  message: string;
  updated: string[];  // 更新された設定キー
  errors?: {
    key: string;
    error: string;
  }[];
}
```

#### 処理フロー
1. パスワード認証確認（Cookie: config_auth）
2. リクエストボディの解析とバリデーション
3. 各設定項目のバリデーション実行
4. ConfigService.setAll()で設定更新
5. 更新結果のレスポンス

#### JavaScript機能
```javascript
// 個別設定の保存
async function saveConfig(key) {
  const input = document.querySelector(`input[name="${key}"]`);
  const response = await fetch('/api/v1/config/update', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      updates: [{
        key: key,
        value: input.value,
        type: input.dataset.type || 'string'
      }]
    })
  });
  
  const result = await response.json();
  showNotification(result.success, result.message);
}

// すべての設定の一括保存
async function saveAllConfigs() {
  const inputs = document.querySelectorAll('.config-input');
  const updates = Array.from(inputs).map(input => ({
    key: input.name,
    value: input.value,
    type: input.dataset.type || 'string'
  }));
  
  const response = await fetch('/api/v1/config/update', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ updates })
  });
  
  const result = await response.json();
  showNotification(result.success, result.message);
}
```

## バリデーション仕様

### 設定項目別バリデーション
```typescript
interface ConfigValidators {
  'google.client_id': (value: string) => boolean;
  'google.client_secret': (value: string) => boolean;
  'auth0.domain': (value: string) => boolean;
  'auth0.client_id': (value: string) => boolean;
  'auth0.client_secret': (value: string) => boolean;
  'app.config_password': (value: string) => boolean;
  'cache.ttl_minutes': (value: string) => boolean;
}

const validators: ConfigValidators = {
  'google.client_id': (value) => value.length > 0 && value.includes('.googleusercontent.com'),
  'google.client_secret': (value) => value.length >= 24,
  'auth0.domain': (value) => /^[a-zA-Z0-9.-]+\.auth0\.com$/.test(value),
  'auth0.client_id': (value) => value.length >= 32,
  'auth0.client_secret': (value) => value.length >= 64,
  'app.config_password': (value) => value.length >= 8,
  'cache.ttl_minutes': (value) => {
    const num = parseInt(value, 10);
    return !isNaN(num) && num > 0 && num <= 1440; // 1日以内
  }
};
```

## セキュリティ要件

### 認証・認可
1. **パスワード認証必須**
   - Cookie: `config_auth` の確認
   - 未認証時は401エラー

2. **CSRF対策**
   - `X-Requested-With: XMLHttpRequest` ヘッダー必須
   - Origin/Refererヘッダーの検証

3. **入力サニタイゼーション**
   - HTMLエスケープ処理
   - SQLインジェクション対策
   - バリデーション必須

### データ保護
1. **機密情報の取り扱い**
   - クライアントシークレット等は更新時のみ平文表示
   - ログには機密情報を記録しない

2. **変更ログ**
   - 設定変更時のログ記録
   - 変更者・変更時刻・変更内容の記録

## エラーハンドリング

### サーバーサイドエラー
```typescript
try {
  await ConfigService.setAll(validatedConfigs);
  return c.json({
    success: true,
    message: '設定が正常に更新されました',
    updated: Object.keys(validatedConfigs)
  });
} catch (error) {
  console.error('Config update error:', error);
  return c.json({
    success: false,
    message: '設定の更新中にエラーが発生しました'
  }, 500);
}
```

### クライアントサイドエラー
```javascript
async function saveConfig(key) {
  try {
    const response = await fetch('/api/v1/config/update', {
      method: 'POST',
      // ...
    });
    
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}: ${response.statusText}`);
    }
    
    const result = await response.json();
    showNotification(result.success, result.message);
  } catch (error) {
    showNotification(false, `エラーが発生しました: ${error.message}`);
  }
}
```
