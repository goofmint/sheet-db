## 概要
Google Sheetsのデータアクセスにおける行レベルでのアクセス制御を提供するACLサービスの実装。各行に設定されたACL情報とConfigテーブルのAPI制御設定に基づいて、ユーザー、ロール、公開アクセスの権限を統一的に管理する。

## 技術要件

### ACLサービス仕様
- **ファイル**: `src/services/acl.ts`
- **役割**: 行レベルアクセス制御とAPI操作制御
- **依存関係**: SessionService、ConfigService、Google Sheets API

### 主要機能

#### checkRowAccess
```typescript
static async checkRowAccess(
  sheetName: string,
  rowData: SheetRow,
  operation: 'read' | 'write',
  context: ACLContext
): Promise<ACLResult>
```
- 行レベルでのアクセス制御チェック
- 行のACL設定（public_read, public_write, user_read, user_write, role_read, role_write）を評価
- ユーザー認証状態とロール情報の確認

#### checkAPIOperation
```typescript
static async checkAPIOperation(
  operation: 'create' | 'modify' | 'delete',
  sheetName: string,
  context: ACLContext
): Promise<ACLResult>
```
- シート操作API（作成・変更・削除）の制御
- Configテーブルの設定値を確認
- api.sheet.allow_create, api.sheet.allow_modify, api.sheet.allow_deleteの評価
- **重要制限**: `_`で始まるシート名（`_User`, `_File`など）は操作禁止（システム予約シート）

#### validateMasterKey
```typescript
static async validateMasterKey(providedKey: string): Promise<boolean>
```
- マスターキー認証の検証
- Configテーブルの`api.master_key_hash`（ハッシュ化済み）と比較
- 提供されたキーをハッシュ化してから定数時間比較
- 全権限でのアクセス許可（ACL設定を無視）
- **セキュリティ**: 生のキーはメモリから即座にクリア、ハッシュ化済み値のみ保存
- **認証方法**: HTTPヘッダー `x-master-key` にキーを設定

#### getUserRoles
```typescript
static async getUserRoles(userId: string): Promise<string[]>
```
- ユーザーのロール情報を取得
- _Roleシートからユーザーが所属するロールを検索
- ロールベースアクセス制御のための基盤機能
- **重要**: 同一リクエスト内でのキャッシュ機能実装が必須

### アクセス制御ロジック

#### 行アクセス制御フロー
1. **マスターキーチェック**: 提供されたキーをハッシュ化してConfigテーブルの`api.master_key_hash`と定数時間比較、一致する場合は全権限を許可
2. **公開アクセスチェック**: 
   - 読み取り: `public_read = true` なら許可
   - 書き込み: `public_write = true` なら許可
3. **認証済みユーザーチェック**:
   - 読み取り: `user_read`配列にuser_idが含まれていれば許可
   - 書き込み: `user_write`配列にuser_idが含まれていれば許可
4. **ロールベースチェック**:
   - 読み取り: ユーザーのロールが`role_read`配列に含まれていれば許可
   - 書き込み: ユーザーのロールが`role_write`配列に含まれていれば許可

#### API操作制御フロー
1. **システムシートチェック**: `_`で始まるシート名の場合は即座に拒否（システム予約シート保護）
2. **マスターキーチェック**: 提供されたキーをハッシュ化してConfigテーブルの`api.master_key_hash`と定数時間比較、一致する場合は全操作を許可
3. **Config設定チェック**:
   - シート作成: `api.sheet.allow_create = true`なら許可
   - シート変更: `api.sheet.allow_modify = true`なら許可
   - シート削除: `api.sheet.allow_delete = true`なら許可
4. **認証状態チェック**: 操作には認証が必要（設定により変更可能）

### Config設定による制御

#### api.master_key_hash
- **型**: string
- **デフォルト**: null (未設定)
- **用途**: 全権限でのアクセスを許可するマスターキーのハッシュ値
- **影響**: 全てのAPI操作とデータアクセス
- **セキュリティ**: SHA-256 + saltを使用したハッシュ化、生のキーは保存しない
- **生成**: システム初期化時に生のキーからハッシュを生成して保存

#### api.master_key_salt
- **型**: string
- **デフォルト**: null (未設定)
- **用途**: マスターキーハッシュ化時のsalt値
- **セキュリティ**: ランダムなUUID、ハッシュ計算時に使用

#### api.sheet.allow_create
- **型**: boolean
- **デフォルト**: false
- **用途**: API経由でのシート作成を許可するかどうか
- **影響**: POST /api/v1/sheets エンドポイント

#### api.sheet.allow_modify
- **型**: boolean
- **デフォルト**: false  
- **用途**: API経由でのシート構造変更を許可するかどうか
- **影響**: シートヘッダー変更、列追加・削除など

#### api.sheet.allow_delete
- **型**: boolean
- **デフォルト**: false
- **用途**: API経由でのシート削除を許可するかどうか
- **影響**: DELETE /api/v1/sheets/:sheetName エンドポイント

### Configテーブル初期化処理

システム初期化時にConfigテーブルにマスターキー関連設定を自動追加する処理：

```typescript
// Configテーブル初期化時の処理（ConfigServiceまたはSetup処理）
async function initializeMasterKeyConfig(): Promise<void> {
  try {
    // 既存のハッシュ設定をチェック
    const existingHash = await ConfigService.get('api.master_key_hash');
    if (existingHash) {
      console.log('Master key configuration already exists');
      return;
    }

    // 初期マスターキーハッシュとsaltを生成
    const salt = crypto.randomUUID();
    
    // Configテーブルに初期値として追加
    const initialConfigs = [
      {
        key: 'api.master_key_hash',
        value: null, // 初期値はnull、POST APIで設定される
        type: 'string',
        description: 'Master key hash for full API access'
      },
      {
        key: 'api.master_key_salt',
        value: salt, // saltは初期化時に生成
        type: 'string',
        description: 'Salt for master key hashing'
      }
    ];

    for (const config of initialConfigs) {
      await ConfigService.set(config.key, config.value, config.type, config.description);
    }

    console.log('Master key configuration initialized');
  } catch (error) {
    console.error('Failed to initialize master key configuration:', error);
    throw error;
  }
}
```

**POST /api/v1/configs での特別処理:**

```typescript
// POST /api/v1/configs エンドポイントでの api.master_key 処理
if (key === 'api.master_key') {
  // 生のマスターキーが送信された場合の特別処理
  const salt = await ConfigService.get('api.master_key_salt');
  if (!salt) {
    throw new Error('Master key salt not found. System initialization required.');
  }
  
  // ハッシュ化処理
  const hash = await hashMasterKey(value, salt);
  
  // ハッシュ値を保存（生のキーは保存しない）
  await ConfigService.set('api.master_key_hash', hash);
  
  // セキュリティ: 生のキーを即座にクリア
  value = '';
  
  return {
    success: true,
    message: 'Master key has been securely hashed and stored',
    key: 'api.master_key_hash'
  };
}
```

### 実装詳細

#### マスターキー管理
- Configテーブルでのハッシュ値の安全な保存（`api.master_key_hash`）
- ConfigServiceを通じたハッシュ値取得
- 提供されたキーのハッシュ化と定数時間比較による認証処理
- 生のキーは即座にメモリから削除
- キー未設定時の適切なハンドリング

**実装例:**
```typescript
static async validateMasterKey(providedKey: string): Promise<boolean> {
  try {
    // Configからハッシュ値を取得
    const storedHash = await ConfigService.get('api.master_key_hash');
    if (!storedHash) {
      return false; // マスターキー未設定
    }
    
    // 提供されたキーをハッシュ化（同じsalt使用）
    const salt = await ConfigService.get('api.master_key_salt');
    const providedHash = await this.hashMasterKey(providedKey, salt);
    
    // 定数時間比較
    const isValid = constantTimeEquals(providedHash, storedHash);
    
    // セキュリティ: 生のキーを即座にクリア
    providedKey = '';
    
    return isValid;
  } catch (error) {
    console.error('Master key validation error:', error);
    return false;
  }
}

private static async hashMasterKey(key: string, salt: string): Promise<string> {
  const encoder = new TextEncoder();
  const keyData = encoder.encode(key + salt);
  const hashBuffer = await crypto.subtle.digest('SHA-256', keyData);
  return Array.from(new Uint8Array(hashBuffer))
    .map(b => b.toString(16).padStart(2, '0'))
    .join('');
}
```

#### ロールキャッシュ実装
```typescript
class ACLService {
  // リクエストスコープキャッシュ（WeakMapで自動ガベージコレクション）
  private static requestCaches = new WeakMap<object, Map<string, string[]>>();
  
  static async getUserRoles(userId: string, context?: object): Promise<string[]> {
    // リクエストコンテキストでキャッシュを管理
    if (context && this.requestCaches.has(context)) {
      const cache = this.requestCaches.get(context)!;
      if (cache.has(userId)) {
        return cache.get(userId)!;
      }
    }
    
    // _Roleシートからユーザーが所属するロールを検索
    const roles = await this.fetchUserRolesFromRoleSheet(userId);
    
    // リクエストスコープキャッシュに保存
    if (context) {
      if (!this.requestCaches.has(context)) {
        this.requestCaches.set(context, new Map());
      }
      this.requestCaches.get(context)!.set(userId, roles);
    }
    
    return roles;
  }
  
  private static async fetchUserRolesFromRoleSheet(userId: string): Promise<string[]> {
    // _Roleシートの全行を取得
    const roleSheet = new RoleSheet(this.env);
    const rolesResult = await roleSheet.findAll();
    
    if (!rolesResult.success || !rolesResult.data) {
      return [];
    }
    
    const userRoles: string[] = [];
    
    // 各ロールのusers配列をチェック
    for (const role of rolesResult.data) {
      try {
        const users = Array.isArray(role.users) ? role.users : JSON.parse(role.users || '[]');
        if (users.includes(userId)) {
          userRoles.push(role.id);
        }
      } catch (error) {
        console.warn(`Failed to parse users for role ${role.id}:`, error);
      }
    }
    
    return userRoles;
  }
  
  // WeakMapのため明示的なクリアは不要（リクエスト終了時に自動削除）
  static clearRequestCache(context: object): void {
    this.requestCaches.delete(context);
  }
}
```

#### セキュリティ考慮事項
- マスターキーの定期的な変更推奨（ハッシュ値とsaltも更新）
- ログでの生のマスターキー値の完全非表示
- 提供されたキーはvalidation後に即座にメモリからクリア
- 不正使用試行の監視とアラート
- HTTPS通信でのキー送信必須
- **ハッシュ化**: SHA-256 + saltを使用、生のキーは一切保存しない
- **システムシート保護**: `_`で始まるシート名（`_User`, `_File`など）の操作を完全禁止
- システムシート操作試行の監査ログ記録
- メモリリーク対策: キー使用後の即座なクリア処理
