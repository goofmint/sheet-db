# 2.4.1 ACL Service - アクセス制御サービス

## ステータス
- [ ] 未完了
- 開始日: 2025-08-03
- 完了日: -

## 概要
Google Sheetsのデータアクセスにおける詳細なアクセス制御を提供するACLサービスの実装。シート、行、列レベルでの権限管理を行い、ユーザー、ロール、公開アクセスの制御を統一的に管理する。

## 技術要件

### ACLサービス仕様
- **ファイル**: `src/services/acl.ts`
- **役割**: 詳細なアクセス制御とパーミッション管理
- **依存関係**: SessionService、UserSheet、ConfigService

### 主要機能

#### checkAccess
```typescript
static async checkAccess(
  resource: ACLResource,
  operation: ACLOperation,
  context: ACLContext
): Promise<ACLResult>
```
- リソースへのアクセス権限をチェック
- ユーザー、ロール、公開アクセスの統合判定
- 詳細なアクセス制御ルールの適用

#### checkSheetAccess
```typescript
static async checkSheetAccess(
  sheetName: string,
  operation: ACLOperation,
  context: ACLContext
): Promise<ACLResult>
```
- シートレベルでのアクセス制御
- シート固有のACL設定の確認
- 継承されたアクセス権限の処理

#### checkRowAccess
```typescript
static async checkRowAccess(
  sheetName: string,
  rowId: string,
  operation: ACLOperation,
  context: ACLContext
): Promise<ACLResult>
```
- 行レベルでのアクセス制御
- ユーザー所有権の確認
- 動的アクセス制御ルールの適用

#### validateMasterKey
```typescript
static validateMasterKey(providedKey: string): boolean
```
- マスターキー認証の検証
- 全権限でのアクセス許可
- セキュアなキー比較処理

### データ型定義

#### ACLResource
```typescript
interface ACLResource {
  type: 'sheet' | 'row' | 'column' | 'cell';
  sheet_name: string;
  row_id?: string;
  column_name?: string;
  cell_ref?: string;
}
```

#### ACLOperation
```typescript
type ACLOperation = 'read' | 'write' | 'delete' | 'create' | 'admin';
```

#### ACLContext
```typescript
interface ACLContext {
  user_id?: string;
  session_id?: string;
  master_key?: string;
  roles?: string[];
  is_public?: boolean;
  request_origin?: string;
}
```

#### ACLResult
```typescript
interface ACLResult {
  allowed: boolean;
  reason?: string;
  required_permission?: string;
  fallback_access?: boolean;
}
```

#### ACLRule
```typescript
interface ACLRule {
  id: string;
  resource: ACLResource;
  permissions: {
    public_read: boolean;
    public_write: boolean;
    user_read: string[];
    user_write: string[];
    role_read: string[];
    role_write: string[];
  };
  conditions?: ACLCondition[];
  priority: number;
}
```

#### ACLCondition
```typescript
interface ACLCondition {
  type: 'time' | 'ip' | 'origin' | 'custom';
  operator: 'equals' | 'contains' | 'matches' | 'between';
  value: string | string[] | number[];
}
```

### アクセス制御レベル

#### 1. マスターキーアクセス
- **最高権限**: 全てのリソースに対する完全なアクセス
- **用途**: 管理操作、システムメンテナンス
- **認証**: 環境変数で設定されたマスターキー

#### 2. 認証済みユーザーアクセス
- **ユーザー固有権限**: 個別ユーザーへの権限付与
- **ロールベース権限**: ユーザーロールによる権限継承
- **所有者権限**: データ作成者による特別な権限

#### 3. 公開アクセス
- **読み取り専用**: 公開設定されたデータの閲覧
- **制限付きアクセス**: 特定条件下での限定的なアクセス
- **ゲストアクセス**: 未認証ユーザーでの基本操作

### 使用例

#### シート読み取り権限チェック
```typescript
const context: ACLContext = {
  user_id: 'auth0|user123',
  session_id: 'session456'
};

const result = await ACLService.checkSheetAccess(
  'users',
  'read',
  context
);

if (!result.allowed) {
  throw new Error(`Access denied: ${result.reason}`);
}
```

#### マスターキー認証
```typescript
const isMasterKeyValid = ACLService.validateMasterKey(
  request.headers['x-master-key']
);

if (isMasterKeyValid) {
  // 全権限でのアクセス許可
}
```
