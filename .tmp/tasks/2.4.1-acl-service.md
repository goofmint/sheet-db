# 2.4.1 ACL Service - アクセス制御サービス

## 概要
Google Sheetsのデータアクセスにおける行レベルでのアクセス制御を提供するACLサービスの実装。各行に設定されたACL情報とConfigテーブルのAPI制御設定に基づいて、ユーザー、ロール、公開アクセスの権限を統一的に管理する。

## 技術要件

### ACLサービス仕様
- **ファイル**: `src/services/acl.ts`
- **役割**: 行レベルアクセス制御とAPI操作制御
- **依存関係**: SessionService、ConfigService、Google Sheets API

### 主要機能

#### checkRowAccess
```typescript
static async checkRowAccess(
  sheetName: string,
  rowData: SheetRow,
  operation: 'read' | 'write',
  context: ACLContext
): Promise<ACLResult>
```
- 行レベルでのアクセス制御チェック
- 行のACL設定（public_read, public_write, user_read, user_write, role_read, role_write）を評価
- ユーザー認証状態とロール情報の確認

#### checkAPIOperation
```typescript
static async checkAPIOperation(
  operation: 'create' | 'modify' | 'delete',
  context: ACLContext
): Promise<ACLResult>
```
- シート操作API（作成・変更・削除）の制御
- Configテーブルの設定値を確認
- api.sheet.allow_create, api.sheet.allow_modify, api.sheet.allow_deleteの評価

#### validateMasterKey
```typescript
static validateMasterKey(providedKey: string): boolean
```
- マスターキー認証の検証
- 全権限でのアクセス許可（ACL設定を無視）
- セキュアな定数時間比較処理

#### getUserRoles
```typescript
static async getUserRoles(userId: string): Promise<string[]>
```
- ユーザーのロール情報を取得
- _Userシートまたは外部システムからロール情報を取得
- ロールベースアクセス制御のための基盤機能

### データ型定義

#### ACLContext
```typescript
interface ACLContext {
  user_id?: string;
  session_id?: string;
  master_key?: string;
  is_authenticated: boolean;
}
```

#### ACLResult
```typescript
interface ACLResult {
  allowed: boolean;
  reason?: string;
  fallback_to_public?: boolean;
}
```

#### RowACLData
```typescript
interface RowACLData {
  public_read: boolean;
  public_write: boolean;
  user_read: string[];      // User IDの配列
  user_write: string[];     // User IDの配列
  role_read: string[];      // Role名の配列
  role_write: string[];     // Role名の配列
}
```

### アクセス制御ロジック

#### 行アクセス制御フロー
1. **マスターキーチェック**: 提供されている場合、全権限を許可
2. **公開アクセスチェック**: 
   - 読み取り: `public_read = true` なら許可
   - 書き込み: `public_write = true` なら許可
3. **認証済みユーザーチェック**:
   - 読み取り: `user_read`配列にuser_idが含まれていれば許可
   - 書き込み: `user_write`配列にuser_idが含まれていれば許可
4. **ロールベースチェック**:
   - 読み取り: ユーザーのロールが`role_read`配列に含まれていれば許可
   - 書き込み: ユーザーのロールが`role_write`配列に含まれていれば許可

#### API操作制御フロー
1. **マスターキーチェック**: 提供されている場合、全操作を許可
2. **Config設定チェック**:
   - シート作成: `api.sheet.allow_create = true`なら許可
   - シート変更: `api.sheet.allow_modify = true`なら許可
   - シート削除: `api.sheet.allow_delete = true`なら許可
3. **認証状態チェック**: 操作には認証が必要（設定により変更可能）

### Config設定による制御

#### api.sheet.allow_create
- **型**: boolean
- **デフォルト**: false
- **用途**: API経由でのシート作成を許可するかどうか
- **影響**: POST /api/v1/sheets エンドポイント

#### api.sheet.allow_modify
- **型**: boolean
- **デフォルト**: false  
- **用途**: API経由でのシート構造変更を許可するかどうか
- **影響**: シートヘッダー変更、列追加・削除など

#### api.sheet.allow_delete
- **型**: boolean
- **デフォルト**: false
- **用途**: API経由でのシート削除を許可するかどうか
- **影響**: DELETE /api/v1/sheets/:sheetName エンドポイント

### 使用例

#### 行データ読み取り権限チェック
```typescript
const context: ACLContext = {
  user_id: 'auth0|user123',
  session_id: 'session456',
  is_authenticated: true
};

const rowData = {
  id: 'row1',
  name: 'John Doe',
  public_read: false,
  user_read: ['auth0|user123'],
  role_read: ['admin']
  // ...other columns
};

const result = await ACLService.checkRowAccess(
  'users',
  rowData,
  'read',
  context
);

if (!result.allowed) {
  // アクセス拒否処理
}
```

#### API操作権限チェック
```typescript
const context: ACLContext = {
  user_id: 'auth0|user123',
  is_authenticated: true
};

const result = await ACLService.checkAPIOperation(
  'create',
  context
);

if (!result.allowed) {
  return res.status(403).json({
    success: false,
    error: 'sheet_creation_disabled'
  });
}
```

#### マスターキー認証
```typescript
const masterKey = request.headers['x-master-key'];
const isMasterKeyValid = ACLService.validateMasterKey(masterKey);

if (isMasterKeyValid) {
  // 全権限でのアクセス許可、ACLチェックをスキップ
  return { allowed: true };
}
```
