## 概要
Auth0統合のためのサービスクラスを作成します。このサービスは、Auth0との認証フロー、トークン検証、ユーザー情報取得などの機能を提供します。

## 実装内容

### 1. Auth0サービスクラス (`src/services/auth0.ts`)

#### 主要機能
1. **認証URLの生成**
   - OAuth2.0認可コードフローに基づいたAuth0ログインURLの生成
   - state/nonceパラメータによるCSRF対策
   - リダイレクトURI設定

2. **トークン交換**
   - 認可コードをアクセストークンに交換
   - リフレッシュトークンの取得
   - トークン有効期限の管理

3. **トークン検証**
   - JWTトークンの署名検証
   - トークン有効期限チェック
   - audience/issuer検証

4. **ユーザー情報取得**
   - Auth0 UserInfo APIの呼び出し
   - ユーザープロファイル情報の取得
   - emailやサブジェクトIDの抽出

5. **リフレッシュトークン処理**
   - アクセストークン更新
   - リフレッシュトークンローテーション

#### 設計方針
- **Configテーブルから設定取得**: ConfigServiceを通じてAuth0設定を取得（環境変数は使用しない）
- **エラーハンドリング**: Auth0 API エラーの適切な処理
- **型安全性**: TypeScriptインターフェースの活用
- **セキュリティ**: PKCE (Proof Key for Code Exchange) の実装検討

### 2. 依存関係
- `src/services/config.ts` - Auth0設定の取得
- `src/types/session.d.ts` - Auth0Profile, AuthToken型定義
- `src/utils/security.ts` - トークン検証用ユーティリティ

### 3. Auth0設定項目
```typescript
interface Auth0Config {
  domain: string;        // Auth0テナントドメイン
  clientId: string;      // アプリケーションのクライアントID
  clientSecret: string;  // アプリケーションのクライアントシークレット
  redirectUri: string;   // コールバックURL
  audience?: string;     // API識別子（オプション）
  scope: string;        // 要求するスコープ（デフォルト: 'openid profile email'）
}
```

### 4. 実装例
```typescript
export class Auth0Service {
  constructor(
    private configService: ConfigService,
    private env: Env
  ) {}

  // 認証URLの生成
  async getAuthorizationUrl(state: string, redirectUri: string): Promise<string>

  // トークン交換
  async exchangeCodeForToken(code: string, state: string): Promise<AuthToken>

  // トークン検証
  async verifyToken(token: string): Promise<Auth0Profile>

  // ユーザー情報取得
  async getUserInfo(accessToken: string): Promise<Auth0Profile>

  // リフレッシュトークンでの更新
  async refreshAccessToken(refreshToken: string): Promise<AuthToken>
}
```

## メモ
- Auth0の無料プランでの制限事項を考慮
- JWKS (JSON Web Key Set) エンドポイントからの公開鍵取得実装
- Rate Limiting対策の実装
- エラーレスポンスの標準化（Auth0エラーコードのマッピング）
- **重要: モック実装は絶対禁止** - テストでも実際のAuth0サービスを使用すること
- **テスト用認証情報**: AUTH0_TEST_EMAIL と AUTH0_TEST_PASSWORD を使用（.dev.vars / .test.vars / .env に設定済み）
