# タスク1.2: D1データベーススキーマの実装

## 概要

Cloudflare D1 SQLiteデータベースのスキーマ設計と実装を行います。このタスクでは、GoogleシートをバックエンドDBとして機能させるために必要な3つの主要テーブル（Config、Cache、Session）を定義し、Drizzle ORMを使用してタイプセーフなデータベース操作を可能にします。

## 目標

1. **データベーススキーマの設計**
   - Config: アプリケーション設定情報の管理
   - Cache: Googleシートデータのキャッシュ
   - Session: ユーザーセッション管理

2. **Drizzle ORM統合**
   - TypeScriptでのタイプセーフなDB操作
   - マイグレーション管理
   - リポジトリパターンの実装

3. **データベース操作の基盤構築**
   - CRUD操作のベース実装
   - エラーハンドリング
   -接続管理

## データベース設計

### Configテーブル
アプリケーション全体の設定情報を管理
```sql
CREATE TABLE Config (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    key TEXT NOT NULL UNIQUE,
    value TEXT NOT NULL,
    type TEXT NOT NULL DEFAULT 'string', -- string, number, boolean, json
    description TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

**用途:**
- Google認証設定
- キャッシュ設定（有効期限など）
- API設定
- UI設定（セットアップ完了フラグなど）

### Cacheテーブル
Googleシートデータのキャッシュを管理
```sql
CREATE TABLE Cache (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    cache_key TEXT NOT NULL UNIQUE,
    data TEXT NOT NULL, -- JSON形式のデータ
    expires_at DATETIME NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    metadata TEXT -- JSON形式のメタデータ（スキーマ情報など）
);
```

**用途:**
- Googleシートデータのキャッシュ
- APIレスポンスの一時保存
- スキーマ情報のキャッシュ
- パフォーマンス最適化

### Sessionテーブル
ユーザーセッション情報を管理
```sql
CREATE TABLE Session (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL UNIQUE,
    user_id TEXT NOT NULL,
    user_data TEXT NOT NULL, -- JSON形式のユーザー情報
    access_token TEXT,
    refresh_token TEXT,
    expires_at DATETIME NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

**用途:**
- Auth0セッション管理
- Googleアクセストークン保存
- ユーザー情報キャッシュ
- 認証状態管理

## Drizzle ORM スキーマ定義

### 型定義の方針
```typescript
// 基本的な型
export type ConfigType = 'string' | 'number' | 'boolean' | 'json';

// Config操作用の型
export interface ConfigInsert {
  key: string;
  value: string;
  type?: ConfigType;
  description?: string;
}

export interface ConfigUpdate {
  value?: string;
  type?: ConfigType;
  description?: string;
}

// Cache操作用の型
export interface CacheInsert {
  cache_key: string;
  data: string;
  expires_at: Date;
  metadata?: string;
}

// Session操作用の型
export interface SessionInsert {
  session_id: string;
  user_id: string;
  user_data: string;
  access_token?: string;
  refresh_token?: string;
  expires_at: Date;
}
```

## リポジトリパターン設計

### 共通インターフェース
```typescript
interface BaseRepository<T, InsertT, UpdateT> {
  findById(id: number): Promise<T | null>;
  findAll(limit?: number, offset?: number): Promise<T[]>;
  create(data: InsertT): Promise<T>;
  update(id: number, data: UpdateT): Promise<T | null>;
  delete(id: number): Promise<boolean>;
}
```

### Config Repository
```typescript
interface ConfigRepository extends BaseRepository<Config, ConfigInsert, ConfigUpdate> {
  findByKey(key: string): Promise<Config | null>;
  upsertByKey(key: string, data: ConfigInsert): Promise<Config>;
  deleteByKey(key: string): Promise<boolean>;
  
  // 便利メソッド
  getString(key: string, defaultValue?: string): Promise<string>;
  getNumber(key: string, defaultValue?: number): Promise<number>;
  getBoolean(key: string, defaultValue?: boolean): Promise<boolean>;
  getJson<T>(key: string, defaultValue?: T): Promise<T>;
}
```

### Cache Repository
```typescript
interface CacheRepository extends BaseRepository<Cache, CacheInsert, CacheUpdate> {
  findByKey(cacheKey: string): Promise<Cache | null>;
  findValidByKey(cacheKey: string): Promise<Cache | null>; // 有効期限チェック付き
  upsertByKey(cacheKey: string, data: CacheInsert): Promise<Cache>;
  deleteByKey(cacheKey: string): Promise<boolean>;
  deleteExpired(): Promise<number>; // 期限切れキャッシュの削除
  
  // キャッシュ操作
  get<T>(key: string): Promise<T | null>;
  set<T>(key: string, data: T, expiresAt: Date, metadata?: object): Promise<void>;
  invalidate(key: string): Promise<void>;
  clear(): Promise<void>;
}
```

### Session Repository
```typescript
interface SessionRepository extends BaseRepository<Session, SessionInsert, SessionUpdate> {
  findBySessionId(sessionId: string): Promise<Session | null>;
  findValidBySessionId(sessionId: string): Promise<Session | null>; // 有効期限チェック付き
  findByUserId(userId: string): Promise<Session[]>;
  deleteBySessionId(sessionId: string): Promise<boolean>;
  deleteExpired(): Promise<number>; // 期限切れセッションの削除
  
  // セッション操作
  createSession(userId: string, userData: object, expiresAt: Date): Promise<Session>;
  updateTokens(sessionId: string, accessToken?: string, refreshToken?: string): Promise<Session | null>;
  extendExpiry(sessionId: string, newExpiresAt: Date): Promise<Session | null>;
}
```

## マイグレーション戦略

### 1. 初期マイグレーション
- 3つのテーブル作成

### 2. インデックス設計
```sql
-- Configテーブル
CREATE INDEX idx_config_key ON Config(key);

-- Cacheテーブル
CREATE INDEX idx_cache_key ON Cache(cache_key);
CREATE INDEX idx_cache_expires_at ON Cache(expires_at);

-- Sessionテーブル
CREATE INDEX idx_session_session_id ON Session(session_id);
CREATE INDEX idx_session_user_id ON Session(user_id);
CREATE INDEX idx_session_expires_at ON Session(expires_at);
```

### 3. トリガー設計
```sql
-- updated_at自動更新
CREATE TRIGGER update_config_updated_at 
    AFTER UPDATE ON Config
    BEGIN
        UPDATE Config SET updated_at = CURRENT_TIMESTAMP WHERE id = NEW.id;
    END;

CREATE TRIGGER update_cache_updated_at 
    AFTER UPDATE ON Cache
    BEGIN
        UPDATE Cache SET updated_at = CURRENT_TIMESTAMP WHERE id = NEW.id;
    END;

CREATE TRIGGER update_session_updated_at 
    AFTER UPDATE ON Session
    BEGIN
        UPDATE Session SET updated_at = CURRENT_TIMESTAMP WHERE id = NEW.id;
    END;
```

## エラーハンドリング設計

### データベースエラーの分類
1. **接続エラー**: D1データベースへの接続失敗
2. **制約違反**: UNIQUE制約などの違反
3. **データ不整合**: 期待する形式でないデータ
4. **トランザクションエラー**: 複数操作での整合性エラー

### エラーレスポンス形式
```typescript
export interface DatabaseError extends Error {
  code: string;
  table?: string;
  operation?: 'SELECT' | 'INSERT' | 'UPDATE' | 'DELETE';
  constraint?: string;
}
```
