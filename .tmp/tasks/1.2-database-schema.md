# タスク1.2: D1データベーススキーマの実装

## 概要

Cloudflare D1 SQLiteデータベースのスキーマ設計と実装を行います。このタスクでは、GoogleシートをバックエンドDBとして機能させるために必要な3つの主要テーブル（Config、Cache、Session）を定義し、Drizzle ORMを使用してタイプセーフなデータベース操作を可能にします。

## 目標

1. **データベーススキーマの設計**
   - Config: アプリケーション設定情報の管理
   - Cache: Googleシートデータのキャッシュ
   - Session: ユーザーセッション管理

2. **Drizzle ORM統合**
   - TypeScriptでのタイプセーフなDB操作
   - マイグレーション管理
   - リポジトリパターンの実装

3. **データベース操作の基盤構築**
   - CRUD操作のベース実装
   - エラーハンドリング
   - 接続管理

## データベース設計

### Configテーブル
アプリケーション全体の設定情報を管理
```sql
CREATE TABLE Config (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    key TEXT NOT NULL UNIQUE,
    value TEXT NOT NULL,
    type TEXT NOT NULL DEFAULT 'string' CHECK (type IN ('string', 'number', 'boolean', 'json')),
    description TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

**用途:**
- Google認証設定
- キャッシュ設定（有効期限など）
- API設定
- UI設定（セットアップ完了フラグなど）
- シートのアクセス制御（APIでのシート作成許可、不許可など）

### Cacheテーブル
Googleシートデータのキャッシュを管理
```sql
CREATE TABLE Cache (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    cache_key TEXT NOT NULL UNIQUE,
    data TEXT NOT NULL, -- JSON形式のデータ
    expires_at DATETIME NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    metadata TEXT -- JSON形式のメタデータ（スキーマ情報など）
);
```

**用途:**
- Googleシートデータのキャッシュ
- APIレスポンスの一時保存
- スキーマ情報のキャッシュ
- パフォーマンス最適化

### Sessionテーブル
ユーザーセッション情報を管理
```sql
CREATE TABLE Session (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL UNIQUE,
    user_id TEXT NOT NULL,
    user_data TEXT NOT NULL, -- JSON形式のユーザー情報
    access_token TEXT,
    refresh_token TEXT,
    expires_at DATETIME NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

**用途:**
- Auth0セッション管理
- ユーザー情報キャッシュ
- 認証状態管理

## パフォーマンス考慮事項

### 1. Configテーブルのメモリキャッシュ戦略

#### 設計方針
Configテーブルのデータは頻繁にアクセスされるため、アプリケーション起動時に全件をメモリに読み込み、同期的にアクセスできるようにします。

```typescript
class ConfigService {
  private static configCache = new Map<string, Config>();
  private static initialized = false;
  
  // 起動時初期化
  static async initialize(db: Database): Promise<void> {
    const configs = await db.select().from(configTable);
    this.configCache.clear();
    
    for (const config of configs) {
      this.configCache.set(config.key, config);
    }
    
    this.initialized = true;
  }
  
  // 同期的なアクセス
  static getString(key: string, defaultValue = ''): string {
    if (!this.initialized) {
      throw new Error('ConfigService not initialized');
    }
    
    const config = this.configCache.get(key);
    return config?.value ?? defaultValue;
  }
  
  // 設定更新時はDBとキャッシュの両方を更新
  static async upsert(key: string, value: string, type: ConfigType): Promise<void> {
    // DB更新
    const updated = await db.insert(configTable)
      .values({ key, value, type })
      .onConflictDoUpdate({ target: configTable.key, set: { value, type } })
      .returning();
    
    // キャッシュ更新
    this.configCache.set(key, updated[0]);
  }
}
```

#### メリット
1. **高速アクセス**: データベースクエリ不要で即座にアクセス
2. **リクエスト処理の高速化**: 設定値取得によるレイテンシを削減
3. **データベース負荷軽減**: 頻繁なクエリを回避

#### 注意事項
1. **初期化タイミング**: アプリケーション起動時に必ず実行
2. **更新の一貫性**: DB更新とキャッシュ更新は原子的に実行
3. **メモリ使用量**: Configテーブルは小さいため問題なし
