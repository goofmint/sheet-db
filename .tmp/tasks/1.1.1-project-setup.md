# タスク 1.1.1: src/index.tsの作成（メインエントリーポイント）

## 概要

sheetDBプロジェクトのメインエントリーポイントを作成します。

## 実装内容

### メインエントリーポイント (`src/index.ts`)

- Honoアプリケーションの初期化
- 基本的なミドルウェアの設定（CORS、ログ）
- ルートパスのリダイレクト処理（未セットアップ時は`/setup`へ、セットアップ済みは`/playground`へ）
- 各エンドポイントハンドラーのインポートと登録
  - `/health` → `src/api/health/get.ts`
  - `/setup` → hono/jsxで `src/templates/setup.tsx` を読み込み
  - `/playground` → hono/jsxで `src/templates/playground.tsx` を読み込み
- Cloudflare Workers用のfetchハンドラーのエクスポート

## 作成するファイル

1. `src/index.ts` - メインエントリーポイント
2. `src/api/health/get.ts` - ヘルスチェックエンドポイント
3. `src/templates/setup.tsx` - セットアップ画面の最小限テンプレート
4. `src/templates/playground.tsx` - Playground画面の最小限テンプレート

## テスト方針

### 統合テスト

1. **基本的なルーティングテスト** (`test/index.spec.ts`)
   - ルートパスへのアクセス（リダイレクトの確認）
   - ヘルスチェックエンドポイントの動作確認
   - セットアップ画面の表示確認
   - Playground画面の表示確認

## 注意事項

- Cloudflare Workers環境での動作を考慮
- 型定義は最小限（Env型のみ）
- エラーハンドリングミドルウェアは後のタスクで実装するため、ここでは含めない
- セットアップ完了の判定は仮実装（TODO コメントを残す）
- テンプレートは最小限の内容で実装（後のタスクで充実させる）
