## 概要
Auth0からの認証コールバックを処理するAPIエンドポイント `src/api/v1/auth/callback/get.ts` を作成します。このエンドポイントは、Auth0での認証完了後にユーザーがリダイレクトされる先で、認証コードをアクセストークンに交換し、ユーザー情報を取得してJSONレスポンスを返します。

## 実装内容

### 1. コールバックAPIエンドポイント (`src/api/v1/auth/callback/get.ts`)

#### 主要機能
1. **認証コード検証**
   - クエリパラメータから認証コード（`code`）とstate取得
   - CSRFトークン（state）の検証
   - エラーレスポンスの処理

2. **トークン交換処理**
   - Auth0Serviceを使用してコードをアクセストークンに交換
   - IDトークンとアクセストークンの取得
   - トークンの検証

3. **ユーザー情報取得**
   - Auth0からユーザー情報を取得
   - 必要な情報の抽出（user_id、email、name等）

4. **ユーザーデータ管理**
   - _Userシートの更新/作成
   - 新規ユーザーの場合：ユーザーレコード作成
   - 既存ユーザーの場合：最終ログイン時刻更新

5. **セッション管理**
   - セッションテーブルに新しい行を追加
   - セッションID生成とトークン保存
   - セッション有効期限の設定

6. **認証情報のJSONレスポンス**
   - ユーザー情報
   - セッション情報
   - 認証状態

#### API設計

**エンドポイント**: `GET /api/v1/auth/callback`

**クエリパラメータ**:
- `code` (required): Auth0からの認証コード
- `state` (required): CSRF保護用のstateパラメータ
- `error` (optional): エラー時のエラーコード
- `error_description` (optional): エラーの詳細説明

**レスポンス**:
- **成功時**: HTTP 200 with JSON
```json
{
  "success": true,
  "user": {
    "id": "auth0|xxxxxxxxxx",
    "email": "user@example.com",
    "name": "User Name",
    "picture": "https://...",
    "created_at": "2025-07-28T12:00:00Z",
    "last_login": "2025-07-28T12:00:00Z"
  },
  "session": {
    "session_id": "sess_xxxxxxxxxx",
    "expires_at": "2025-07-29T12:00:00Z"
  },
  "authenticated": true
}
```

- **エラー時**: HTTP 400/500 with JSON
```json
{
  "success": false,
  "error": "authentication_failed",
  "message": "詳細なエラーメッセージ",
  "authenticated": false
}
```

#### 実装例
```typescript
import { Hono } from 'hono';
import { Auth0Service } from '../../../../services/auth0';
import { ConfigService } from '../../../../services/config';
import { drizzle } from 'drizzle-orm/d1';
import { sessionTable } from '../../../../db/schema';
import { eq } from 'drizzle-orm';
import { Env } from '../../../../types/env';

const app = new Hono<{ Bindings: Env }>();

app.get('/', async (c) => {
  try {
    // クエリパラメータの取得
    const code = c.req.query('code');
    const state = c.req.query('state');
    const error = c.req.query('error');
    const errorDescription = c.req.query('error_description');

    // Auth0エラーの処理
    if (error) {
      return c.json({
        success: false,
        error: error,
        message: errorDescription || 'Authentication failed',
        authenticated: false
      }, 400);
    }

    // 必須パラメータの検証
    if (!code || !state) {
      return c.json({
        success: false,
        error: 'missing_parameters',
        message: 'Missing required parameters: code and state',
        authenticated: false
      }, 400);
    }

    // State検証（CSRF対策）
    const storedState = c.cookie('auth_state');
    if (!storedState || storedState !== state) {
      return c.json({
        success: false,
        error: 'invalid_state',
        message: 'Invalid state parameter',
        authenticated: false
      }, 400);
    }

    // リダイレクトURIの構築
    const currentHost = new URL(c.req.url).origin;
    const redirectUri = `${currentHost}/api/v1/auth/callback`;

    // Auth0トークン交換
    const auth0Service = new Auth0Service(c.env);
    const tokens = await auth0Service.exchangeCodeForToken(code, redirectUri);

    // ユーザー情報取得
    const userInfo = await auth0Service.getUserInfo(tokens.access_token);

    // データベース接続
    const db = drizzle(c.env.DB);

    // _Userシートの更新/作成（Google Sheets API経由）
    const userData = await upsertUserInSheet(c.env, {
      auth0_id: userInfo.sub,
      email: userInfo.email,
      name: userInfo.name,
      picture: userInfo.picture,
      last_login: new Date().toISOString()
    });

    // セッション作成
    const sessionId = `sess_${crypto.randomUUID()}`;
    const expiresAt = new Date(Date.now() + 24 * 60 * 60 * 1000); // 24時間

    await db.insert(sessionTable).values({
      id: sessionId,
      user_id: userInfo.sub,
      access_token: tokens.access_token,
      id_token: tokens.id_token,
      refresh_token: tokens.refresh_token,
      expires_at: expiresAt.toISOString(),
      created_at: new Date().toISOString()
    });

    // セッションIDをHTTPOnlyクッキーに設定
    c.cookie('session_id', sessionId, {
      httpOnly: true,
      secure: currentHost.startsWith('https'),
      sameSite: 'Lax',
      maxAge: 24 * 60 * 60, // 24時間
      path: '/'
    });

    // Stateクッキー削除
    c.cookie('auth_state', '', { maxAge: -1 });

    // 成功レスポンス
    return c.json({
      success: true,
      user: {
        id: userInfo.sub,
        email: userInfo.email,
        name: userInfo.name,
        picture: userInfo.picture,
        created_at: userData.created_at,
        last_login: userData.last_login
      },
      session: {
        session_id: sessionId,
        expires_at: expiresAt.toISOString()
      },
      authenticated: true
    });

  } catch (error) {
    console.error('Callback error:', error);
    
    // エラーレスポンス
    return c.json({
      success: false,
      error: 'authentication_failed',
      message: 'Authentication process failed',
      authenticated: false
    }, 500);
  }
});
```

### 4. データ管理設計

#### _Userシート構造
```
| auth0_id | email | name | picture | created_at | last_login |
|----------|-------|------|---------|------------|-------------|
| auth0|xxx | user@... | User Name | https://... | 2025-07-28T... | 2025-07-28T... |
```

**カラム説明**:
- `auth0_id`: Auth0ユーザーID（一意キー）
- `email`: ユーザーのメールアドレス
- `name`: ユーザーの表示名
- `picture`: プロフィール画像URL
- `created_at`: ユーザー作成日時（初回ログイン時）
- `last_login`: 最終ログイン日時（毎回更新）

#### セッションテーブル構造（D1）
```sql
CREATE TABLE Session (
  id TEXT PRIMARY KEY,           -- セッションID
  user_id TEXT NOT NULL,        -- Auth0ユーザーID
  access_token TEXT NOT NULL,   -- Auth0アクセストークン
  id_token TEXT,                -- Auth0 IDトークン
  refresh_token TEXT,           -- Auth0リフレッシュトークン
  expires_at TEXT NOT NULL,     -- セッション有効期限
  created_at TEXT NOT NULL,     -- セッション作成日時
  updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);
```

**セッション管理ポリシー**:

## セキュリティ考慮事項

### 1. CSRF攻撃対策
- **問題**: 悪意のあるサイトからの認証フローの誘導
- **対策**: stateパラメータによる検証とワンタイム使用

### 2. 認証コード漏洩対策
- **問題**: URLログやリファラーヘッダーでのコード漏洩
- **対策**: コードの即座な消費とトークン交換

### 3. セッション管理
- **問題**: トークンの永続化とセキュア管理
- **対策**: HTTPOnlyクッキーまたはセキュアセッション
