# 2.2.5 PUT /api/v1/configs/:key - 設定項目の更新

## ステータス
- [ ] 未完了
- 開始日: 2025-08-02
- 完了日: -

## 概要
既存の設定項目を更新するAPIエンドポイントの実装。認証済みユーザーが設定項目の値や属性を変更できるようにする。

## 技術要件

### エンドポイント仕様
- **メソッド**: PUT
- **パス**: `/api/v1/configs/{key}`
- **認証**: 必須（Authorizationヘッダーでconfig_passwordを使用）
- **Content-Type**: application/json

### リクエストボディ
```typescript
interface UpdateConfigRequest {
  value: string | number | boolean | object; // 設定値（必須）
  type: 'string' | 'number' | 'boolean' | 'json'; // 値の型（必須）
  description?: string;  // 説明（任意）
  system_config?: boolean; // システム設定フラグ（任意）
  validation?: {         // バリデーションルール（任意）
    min?: number;        // 最小値（number型の場合）
    max?: number;        // 最大値（number型の場合）
    minLength?: number;  // 最小文字数（string型の場合）
    maxLength?: number;  // 最大文字数（string型の場合）
    pattern?: string;    // 正規表現パターン（string型の場合）
    enum?: (string | number | boolean)[]; // 許可値リスト
    required?: boolean;  // 必須フラグ
  };
}
```

### レスポンス

#### 成功時 (200 OK)
```json
{
  "success": true,
  "data": {
    "id": "d7f3c6e1-3e6b-46cf-a07c-e4d4ac9b7c1d",
    "key": "api_key",
    "value": "sk-updated-key-456",
    "type": "string",
    "description": "Updated API Key for external service",
    "system_config": false,
    "validation": {
      "pattern": "^sk-[a-zA-Z0-9]+$",
      "minLength": 10
    },
    "created_at": "2025-08-02T10:00:00Z",
    "updated_at": "2025-08-02T11:30:00Z"
  }
}
```

#### エラーレスポンス

##### 400 Bad Request - バリデーションエラー
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid configuration data",
    "details": {
      "value": ["Value must be a string for type \"string\""],
      "validation": ["pattern validation is only applicable for string type"]
    }
  }
}
```

##### 401 Unauthorized - 認証エラー
```json
{
  "success": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Authentication required"
  }
}
```

##### 404 Not Found - 設定が存在しない
```json
{
  "success": false,
  "error": {
    "code": "NOT_FOUND",
    "message": "Configuration with key 'nonexistent.key' not found"
  }
}
```

##### 500 Internal Server Error
```json
{
  "success": false,
  "error": {
    "code": "INTERNAL_ERROR",
    "message": "Failed to update configuration"
  }
}
```

## 実装詳細

### バリデーション要件
1. **キーのバリデーション**
   - URLパラメータとして提供
   - 英数字、アンダースコア、ハイフン、ドットのみ許可
   - パターン: `/^[a-zA-Z0-9_.-]+$/`
   - 存在確認（404エラーの場合は適切なレスポンス）

2. **値のバリデーション**
   - 必須項目
   - 型に応じた値の検証
   - JSON型の場合は有効なJSONであることを確認
   - 最大サイズ: 64KB

3. **型のバリデーション**
   - 必須項目
   - 許可値: `string`, `number`, `boolean`, `json`

4. **説明のバリデーション**
   - 任意項目
   - 最大1000文字

5. **バリデーションルールの検証**
   - 指定された型に対して適切なルールか確認
   - 正規表現パターンの妥当性チェック

### セキュリティ考慮事項
1. **入力サニタイゼーション**
   - SQLインジェクション対策（Drizzle ORMが自動的に処理）
   - XSS対策（Zodスキーマによる入力検証）

2. **アクセス制御**
   - Authorizationヘッダーでconfig_passwordによる認証が必須

### データベース操作
1. **トランザクション使用**
   - 設定更新時はトランザクション内で実行
   - エラー時は自動ロールバック

2. **タイムスタンプ**
   - `updated_at`を現在時刻で更新
   - `created_at`は変更しない
