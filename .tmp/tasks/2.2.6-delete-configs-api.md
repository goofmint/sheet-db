# 2.2.6 DELETE /api/v1/configs/:key - 設定項目の削除

## ステータス
- [ ] 未完了
- 開始日: 2025-08-02
- 完了日: -

## 概要
既存の設定項目を削除するAPIエンドポイントの実装。認証済みユーザーが不要になった設定項目を安全に削除できるようにする。

## 技術要件

### エンドポイント仕様
- **メソッド**: DELETE
- **パス**: `/api/v1/configs/{key}`
- **認証**: 必須（Authorizationヘッダーでapp.config_passwordを使用）
- **Content-Type**: application/json（リクエストボディなし）

### パラメータ
- **key** (URL parameter, 必須): 削除する設定項目のキー
  - 英数字、アンダースコア、ハイフン、ドットのみ許可
  - パターン: `/^[a-zA-Z0-9_.-]+$/`

### レスポンス

#### 成功時 (200 OK)
```json
{
  "success": true,
  "data": {
    "message": "Configuration 'api_key' deleted successfully",
    "deleted_key": "api_key"
  }
}
```

#### エラーレスポンス

##### 400 Bad Request - 無効なキー形式
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid configuration key format"
  }
}
```

##### 401 Unauthorized - 認証エラー
```json
{
  "success": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Authentication required"
  }
}
```

##### 404 Not Found - 設定が存在しない
```json
{
  "success": false,
  "error": {
    "code": "NOT_FOUND",
    "message": "Configuration with key 'nonexistent.key' not found"
  }
}
```

##### 403 Forbidden - システム設定の削除禁止
```json
{
  "success": false,
  "error": {
    "code": "FORBIDDEN",
    "message": "Cannot delete system configuration 'app.config_password'"
  }
}
```

##### 500 Internal Server Error
```json
{
  "success": false,
  "error": {
    "code": "INTERNAL_ERROR",
    "message": "Failed to delete configuration"
  }
}
```

## 実装詳細

### バリデーション要件
1. **キーのバリデーション**
   - URLパラメータとして提供
   - 英数字、アンダースコア、ハイフン、ドットのみ許可
   - パターン: `/^[a-zA-Z0-9_.-]+$/`
   - 空文字列やスペースのみは拒否

2. **システム設定保護**
   - 重要なシステム設定（`app.config_password`等）の削除を禁止
   - `system_config = true` の設定項目は削除不可
   - 削除禁止リストによる追加保護

3. **存在確認**
   - 削除前に設定項目の存在を確認
   - 存在しない場合は404エラーを返す

### セキュリティ考慮事項
1. **アクセス制御**
   - Authorizationヘッダーでapp.config_passwordによる認証が必須
   - 認証なしの削除リクエストは拒否

2. **システム設定保護**
   - 重要なシステム設定の誤削除を防止
   - アプリケーション動作に必要な設定の削除を禁止

3. **監査ログ**
   - 削除操作のログ記録
   - 削除されたキーと実行時刻の記録

### データベース操作
1. **トランザクション使用**
   - 設定削除時はトランザクション内で実行
   - エラー時は自動ロールバック

2. **キャッシュ更新**
   - データベースからの削除成功後にメモリキャッシュからも削除
   - キャッシュとDBの一貫性を保証

### 削除禁止設定リスト
以下の設定は削除を禁止する：
- `app.config_password` - 管理画面認証パスワード
- `google.client_id` - Google OAuth設定
- `google.client_secret` - Google OAuth設定
- その他 `system_config = true` の設定
