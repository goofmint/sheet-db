# タスク #2.3: 設定編集機能の仕様策定

## 概要

現在の設定管理インターフェース（`/config`）は読み取り専用で、機密データをマスクして全ての設定項目を表示している。この機能では、セキュリティとデータ整合性を維持しながら、Webインターフェースを通じて設定値を直接変更する機能を追加する。

## セキュリティ要件

### 認証・認可
- **現在のセキュリティ**: HMAC署名付きセッショントークンとdouble-submit-cookie CSRF保護を使用
- **CSRF戦略**: HTTPOnlyクッキーとサーバーサイドトークン検証、JavaScriptからアクセス不可
- **セッション管理**: 2時間のセッションタイムアウトは変更なし
- **アクセス制御**: 有効な設定パスワードを持つユーザーのみが編集機能にアクセス可能

### データ保護
- **機密データ**: 機密値は `input type="password"` で表示（平文で受信するが視覚的に保護）
- **ロールバック保護**: 変更適用前に重要な設定を検証

## ユーザーインターフェース設計

### 現在の状態
- 3つの列を持つ読み取り専用テーブル：設定キー、値、説明
- 機密値は `input type="password"` で表示（APIから平文で受信）
- 入力フィールドは`readonly`で将来の実装を説明するタイトル付き

### 提案する変更

#### 1. 直接編集インターフェース
- **編集ボタン削除**: フォームを直接編集可能にする
- **保存/リセット**: 「すべて保存」と「すべてリセット」ボタンをフォーム下部に配置
- **自動保存**: より良いUXのためのデバウンス付き自動保存（オプション）
- **変更インジケータ**: 変更された項目のリアルタイム表示

#### 3. フィールドタイプと検証

##### テキストフィールド（デフォルト）
- 検証付きの標準テキスト入力
- フィールドタイプに基づく文字数制限
- URL、IDなどのパターン検証

##### パスワード/秘密フィールド
- パスワード入力（`type="password"`）でAPIから受信した平文値を表示
- 「表示/非表示」切り替えボタン
- トークンの自動再生成オプション

##### ブールフィールド
- トグルスイッチまたはチェックボックス
- ラベル付きの明確なオン/オフ状態

##### JSONフィールド
- 構文ハイライト付きコードエディタ（シンプルなtextarea）
- 保存前のJSON検証
- プリティプリント形式

#### 4. 検証とエラーハンドリング

##### クライアントサイド検証
- ユーザーが入力中のリアルタイム検証
- 視覚的インジケータ（緑のチェックマーク、赤のX）
- ツールチップエラーメッセージ
- すべての検証が通るまで保存を無効化

##### サーバーサイド検証
- 各設定タイプのスキーマ検証
- ビジネスロジック検証（例：URLの到達可能性）
- 重複キーの防止
- 必須フィールドの強制

## 機密値の安全な管理

### 機密フィールドのセキュリティ
機密設定値（パスワード、トークン、秘密）は適切なセキュリティで処理：

#### 値の表示
- **パスワードフィールド**: `type="password"` でAPIから受信した平文値を表示
- **視覚的保護**: ブラウザの標準機能で文字がマスクされる
- **編集可能**: 直接値を変更可能
- **表示切り替え**: 必要に応じて平文表示も可能

#### クライアントサイド処理
```javascript
function handleSensitiveField(fieldKey, currentValue) {
  const input = document.getElementById(`input-${fieldKey}`);
  
  // APIから受信した平文値をパスワードフィールドに設定
  if (input.dataset.fieldType === 'sensitive') {
    input.type = 'password';
    input.value = currentValue; // APIから受信した実際の値
    
    // 変更を追跡
    input.addEventListener('input', function() {
      if (this.value !== currentValue) {
        this.closest('.config-row').classList.add('changed');
        showSensitiveChangeWarning(fieldKey);
      } else {
        this.closest('.config-row').classList.remove('changed');
      }
    });
  }
}
```

# 重要

- テストを書く
- テストでスキップを使わない
- コード、コメントはすべて英語
- コードに機密情報を書かない
- TypeScriptのエラーが出ていないこと
- /api/v1/config は不要
- /api/v1/config/auth は GET /api/v1/setupで代替できるので不要
- GET /api/v1/config はHTMLを返しているので不要
- GET /api/v1/config/logout は POST /config/logout で実装すべき
- POST /api/v1/config は POST /api/v1/setup で代替すべき
- GET /api/v1/setup および POST /api/v1/setup で設定を取得・更新する
- src/utils/config-validation.ts は不要。repositories/config-validation.ts で行うべき
- /api/v1 以下はJSONレスポンスを返すこと
- HTML画面は /config 以下で実装する
- HTMLはTSXで作成し、HTMLをコード中に書かない
- クライアントサイドのJavaScript/CSSは別ファイルにする
