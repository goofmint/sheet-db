# [2.2.9] POST /api/v1/configsを利用した/config での設定追加

## ステータス
- [ ] 未完了
- 開始日: 2025-08-04
- 完了日: 

## 概要
/config ページの「Add Configuration」機能において、POST /api/v1/configs APIを使用してクライアントサイドから新しい設定項目を追加できるようにする。

## 実装内容

### 1. JavaScriptクライアント機能の実装
- `/statics/config/client.js`の設定追加機能を実装
- モーダルフォームからの設定追加処理
- フォームバリデーション（クライアントサイド）
- API呼び出しとエラーハンドリング
- 追加成功時のテーブル更新

### 2. フォーム処理の詳細実装
- **入力検証**
  - Key形式の検証（英数字、ドット、アンダースコア）
  - Value形式の検証（型に応じた検証）
  - 必須項目のチェック
  
- **型別処理**
  - `string`: そのまま送信
  - `number`: 数値変換とバリデーション
  - `boolean`: チェックボックス値の変換
  - `json`: JSON文字列の解析と検証

### 3. ユーザーインターフェース機能
- **モーダル制御**
  - 「Add Configuration」ボタンでモーダル表示
  - ESCキー、背景クリックでモーダル閉じる
  - フォーム送信後のモーダル自動クローズ

- **動的フィールド制御**
  - Data Type選択に応じた入力フィールドの切り替え
  - Boolean型選択時はチェックボックス表示
  - JSON型選択時はテキストエリア表示

### 4. API統合とエラーハンドリング
- **POST /api/v1/configs API呼び出し**
  - HttpOnly/SameSiteクッキーによるセッション認証
  - CSRFトークンのヘッダー送信（二重送信クッキーパターン）
  - リクエストボディの適切な構築
  - レスポンス処理（成功/エラー）

- **エラー表示**
  - APIエラーレスポンスの解析
  - ユーザーフレンドリーなエラーメッセージ表示
  - バリデーションエラーの個別表示

### 5. テーブル更新機能
- 設定追加成功時のテーブル行動的追加
- 新しい設定項目の適切なテンプレート適用
- 重複チェックとUI上での表示制御

## 関連ファイル

## 技術的考慮事項

### セキュリティ
- CSRF攻撃対策
  - ランダムなCSRFトークンの生成と管理
  - 二重送信クッキーパターンの実装（クッキー + ヘッダー）
  - SameSite=Lax/Strict、HttpOnly属性の活用
  - Origin/Refererヘッダー確認の併用
- XSS対策（入力値のサニタイズ）
- HttpOnly/SameSiteクッキーによるセッション認証
- config_passwordをクライアントに露出させない設計

### ユーザビリティ
- 直感的なフォーム操作
- 明確なエラーメッセージ
- 操作後の適切なフィードバック

### パフォーマンス
- 不要なDOM操作の最小化
- 効率的なテーブル更新
- 適切なイベントハンドリング

## メモ
- 認証はサーバーサイドで管理し、クライアントには秘密情報を送信しない
- モーダルUIは既に実装済み、JavaScriptロジックの追加が主な作業
- APIレスポンス形式は既存のPOST APIと整合性を保つ
- CSRFトークンはサーバーサイドで生成し、セキュアに管理する
