## 概要
認証済みユーザーの現在の情報を取得するAPIエンドポイントの実装。
認証が必要なAPIで、セッション情報からユーザー情報を返却する。

## API仕様

### エンドポイント
`GET /api/v1/auth/me`

### 目的
- 現在ログイン中のユーザー情報を取得
- フロントエンドでの認証状態確認
- ユーザープロファイル表示用のデータ提供
- セッション有効性の検証

### リクエスト

#### ヘッダー
```
Cookie: session_id=<session_id>
```

#### パラメータ
なし（セッションCookieから認証状態を判定）

### レスポンス仕様

#### 成功時 (200)
```json
{
  "success": true,
  "user": {
    "id": "auth0|user123456789",
    "name": "田中太郎",
    "email": "tanaka@example.com",
    "picture": "https://cdn.auth0.com/avatars/ta.png",
    "email_verified": true,
    "updated_at": "2023-12-01T10:30:00.000Z",
    "iss": "https://your-domain.auth0.com/",
    "aud": "your-client-id",
    "iat": 1701424200,
    "exp": 1701510600,
    "sub": "auth0|user123456789",
    "sid": "session123456789"
  },
  "session": {
    "session_id": "sess_abc123def456",
    "expires_at": "2023-12-02T10:30:00.000Z",
    "created_at": "2023-12-01T10:30:00.000Z"
  }
}
```

#### 認証エラー (401)
```json
{
  "success": false,
  "error": "unauthorized",
  "message": "Authentication required"
}
```

#### セッション期限切れ (401)
```json
{
  "success": false,
  "error": "session_expired",
  "message": "Session has expired"
}
```

#### サーバーエラー (500)
```json
{
  "success": false,
  "error": "server_error",
  "message": "Failed to retrieve user information"
}
```

### 技術仕様

#### 認証フロー
1. リクエストからセッションCookieを取得

3. セッション有効期限を確認
4. ユーザー情報を返却

#### セキュリティ要件
- **HTTPS必須**: 本番環境ではHTTPS接続のみ
- **セッション検証**: 有効なセッションIDの確認
- **期限確認**: expires_atによる有効期限チェック
- **エラー統一**: 攻撃者に情報を与えない一貫したエラーレスポンス

#### パフォーマンス考慮
- **キャッシュなし**: 常に最新のセッション状態を確認
- **データベースアクセス最小化**: 必要な情報のみ取得
- **タイムアウト設定**: 適切なデータベースクエリタイムアウト

## メモ

### 実装時の注意点
1. **エラーレスポンス統一**: セキュリティのため、詳細なエラー情報は含めない
2. **JSON パース**: user_dataフィールドのパース時のエラーハンドリング
3. **タイムゾーン**: expires_atの比較時はUTC時刻を使用
4. **レスポンス形式**: Auth0のユーザー情報形式と一致させる

### Auth0ユーザー情報フィールド説明
- `sub`: Auth0ユーザーID（一意識別子）
- `name`: ユーザーの表示名
- `email`: メールアドレス
- `email_verified`: メール認証状態
- `picture`: プロフィール画像URL
- `updated_at`: 最終更新日時
- `iss`: トークン発行者（Auth0ドメイン）
- `aud`: トークン対象者（クライアントID）
- `iat`: トークン発行時刻
- `exp`: トークン有効期限
- `sid`: セッションID

### セキュリティ考慮事項
1. **セッション固定攻撃対策**: セッションIDの再生成は不要（読み取り専用API）
2. **CSRF対策**: GETリクエストのため不要
3. **XSS対策**: レスポンスデータのサニタイゼーションは不要（JSON形式）
4. **情報漏洩対策**: エラー時は詳細情報を含めない

### パフォーマンス最適化案
1. **セッション情報キャッシュ**: 短時間キャッシュの検討（5-10分）
2. **データベースインデックス**: session_idでの高速検索
3. **レスポンス圧縮**: JSON圧縮の有効化
4. **リクエスト頻度制限**: 同一セッションからの連続リクエスト制限

### 将来の拡張予定
1. **ユーザー設定**: プリファレンス情報の追加
2. **権限情報**: ロール・権限の詳細情報
3. **アクティビティ**: 最終ログイン時刻等
4. **多要素認証**: MFA状態の表示
