## 概要
Auth0認証後のセッション管理を行うSessionServiceの実装。セッション作成、検証、更新、削除の機能を提供し、既存のSessionRepositoryと連携してセッションライフサイクルを管理する。

## 技術要件

### SessionService仕様
- **ファイル**: `src/services/session.ts`
- **役割**: セッション管理のビジネスロジック
- **依存関係**: SessionRepository、Auth0Service

### 主要メソッド

#### createSession
```typescript
static async createSession(userData: Auth0UserData): Promise<SessionCreateResult>
```
- Auth0認証成功後のセッション作成
- ユニークなsession_idの生成
- セッション有効期限の設定（30日）
- 最小限のユーザー識別子のみ保存（PII情報は含まない）
- _Userシートを参照し、データがなければ作成。あれば `last_login` を更新。

#### validateSession
```typescript
static async validateSession(sessionId: string): Promise<SessionValidationResult>
```
- セッションIDの検証
- 有効期限チェック
- 期限切れセッションの自動削除
- ユーザーデータの返却

#### refreshSession
```typescript
static async refreshSession(sessionId: string): Promise<SessionRefreshResult>
```
- セッション有効期限の延長
- アクセス時刻の更新
- 無効セッションのエラーハンドリング

#### deleteSession
```typescript
static async deleteSession(sessionId: string): Promise<boolean>
```
- ログアウト時のセッション削除
- 関連データのクリーンアップ

#### cleanupExpiredSessions
```typescript
static async cleanupExpiredSessions(): Promise<number>
```
- 期限切れセッションの一括削除
- 定期実行用の清掃メソッド

### セッション設定

#### 有効期限管理
- **デフォルト有効期限**: 30日
- **自動延長**: アクセス毎に有効期限をリセット
- **最大セッション時間**: Configテーブルにて、設定可能（デフォルト30日）
  - key: `session_max_age`
  - value: `2592000`（30日）
  - type: `number`（秒単位）
  - default: `2592000`

#### セッションID生成
- **形式**: crypto.randomUUID()
- **長さ**: 36文字（UUID v4）
- **一意性**: データベースレベルで保証

#### セキュリティ考慮事項
- **定数時間比較**: セッションID検証時
- **自動清掃**: 期限切れセッションの削除
- **エラーログ**: セキュリティイベントの記録
- **PII保護**: 最小限の識別子のみ保存、プロファイル情報は遅延取得

#### ユーザープロファイル取得
- **セッション保存**: 最小限の識別子（auth0_user_id, sub）のみ
- **遅延取得**: email、name、pictureは必要時にAuth0APIから取得
- **キャッシュ戦略**: プロファイル情報の一時キャッシュ（別途実装）

### エラーハンドリング

#### セッション作成エラー
- ユーザーデータ不正
- データベース接続エラー
- UUID生成失敗

#### セッション検証エラー
- セッションID不正
- セッション期限切れ
- データベースアクセスエラー

#### セッション削除エラー
- 存在しないセッション
- データベース削除失敗

### 実装パターン

#### シングルトンパターン
- 静的メソッドによるサービス提供
- インスタンス化不要
- グローバルアクセス可能

#### エラーファースト
- すべてのメソッドでエラー優先の戻り値
- 明示的なエラーハンドリング
- ログ出力による追跡可能性

### 統合ポイント

#### Auth0Service連携
- 認証成功後のセッション作成
- ユーザー情報の正規化
- トークン情報の保存

#### Middleware連携
- 認証ミドルウェアでの検証利用
- 自動セッション延長
- リクエスト毎のセッション状態確認

#### API利用
- `/api/v1/auth/me`でのセッション検証
- `/api/v1/auth/logout`でのセッション削除
- 各種認証必須APIでの認証チェック
