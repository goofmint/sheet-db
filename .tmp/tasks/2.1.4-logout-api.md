## 概要
ユーザーのログアウト機能を実装するAPIエンドポイントの作成。セッションの削除、Cookieのクリア、Auth0からのログアウト（オプション）を含む。

## 実装内容

### 1. エンドポイント仕様
- **パス**: `/api/v1/auth/logout`
- **メソッド**: POST
- **認証**: 必須（有効なセッションIDが必要）
- **リクエストボディ**: なし（空のPOSTリクエスト）
- **レスポンス**:
  - 成功時（200）:
    ```json
    {
      "success": true,
      "message": "Successfully logged out"
    }
    ```
  - 認証エラー（401）:
    ```json
    {
      "success": false,
      "error": "unauthorized",
      "message": "No active session found"
    }
    ```

### 2. 処理フロー
1. **セッション検証**
   - CookieからセッションIDを取得
   - D1データベースでセッションの存在と有効性を確認
   - セッションが無効な場合は401エラー

2. **セッション削除**
   - D1データベースからセッションレコードを削除
   - 関連するすべてのセッションデータをクリーンアップ

3. **Cookie削除**
   - session_idクッキーを削除（maxAge: 0）
   - 他の認証関連クッキーがあれば削除

4. **Auth0セッション（考慮事項）**
   - このAPIはローカルセッションのみを削除
   - Auth0側のセッションは残る（シングルサインアウトではない）
   - 完全なログアウトが必要な場合は、クライアント側でAuth0のログアウトURLにリダイレクト

### 3. セキュリティ考慮事項
- **CSRF対策**: 
  - POSTメソッドを使用（GETは使用しない）
  - SameSite=Strict属性のCookie設定を強制
  - Origin/Refererヘッダーの検証実装
  - カスタムCSRFトークンヘッダー（X-Requested-With: XMLHttpRequest）の要求
  - Double Submit Cookieパターンの採用を検討
- **タイミング攻撃対策**: セッションの存在に関わらず一定時間で応答
- **ログ記録**: ログアウトイベントを記録（監査目的）

#### 主要な処理
1. **CSRF検証**:
   ```typescript
   // Origin/Refererヘッダー検証
   const origin = c.req.header('Origin') || c.req.header('Referer');
   const allowedOrigins = [new URL(c.req.url).origin];
   
   // X-Requested-Withヘッダー検証
   const requestedWith = c.req.header('X-Requested-With');
   if (requestedWith !== 'XMLHttpRequest') {
     return c.json({ error: 'Invalid request' }, 400);
   }
   ```

2. **セッション削除クエリ**:
   ```typescript
   await db.delete(sessionTable)
     .where(eq(sessionTable.session_id, sessionId));
   ```

3. **Cookie削除（SameSite=Strict強制）**:
   ```typescript
   deleteCookie(c, 'session_id', {
     httpOnly: true,
     secure: true,
     sameSite: 'Strict', // CSRF対策を強化
     path: '/'
   });
   ```

4. **ログ記録（オプション）**:
   ```typescript
   console.log(`User logged out: ${userId} at ${new Date().toISOString()}`);
   ```

### 5. エラーハンドリング
- セッションが見つからない場合の処理
- データベースエラーのハンドリング
- 無効なセッションIDフォーマット
- CSRF検証失敗（X-Requested-With欠如、Origin/Referer不正）
- 無効なリクエストヘッダー

### 6. テスト項目
- 有効なセッションでのログアウト成功
- 無効なセッションでの401エラー
- セッションCookie削除の確認（SameSite=Strict）
- データベースからのセッション削除確認
- 空のリクエストボディでの正常処理
- CSRF対策の検証:
  - X-Requested-Withヘッダーなしでの400エラー
  - 不正なOrigin/Refererでの400エラー
  - 有効なCSRFヘッダーでの正常処理

## メモ
- 多層のCSRF対策を実装（POST + SameSite=Strict + ヘッダー検証）
- このAPIはローカルセッションのみ削除（Auth0セッションは残る）
- クライアント側で完全ログアウトが必要な場合はAuth0のログアウトURLを使用
- セッション削除は即座に反映される必要がある（キャッシュなし）
- X-Requested-WithヘッダーはAjaxリクエストの標準的なCSRF対策
