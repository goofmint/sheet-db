# 2.3.2 Authentication Middleware - 認証ミドルウェア

- [ ] 未完了

- 完了日: -

## 概要
SessionServiceと連携してHTTPリクエストの認証を行うミドルウェアの実装。Cookieからセッション情報を読み取り、セッションの有効性を検証し、認証が必要なエンドポイントへのアクセス制御を提供する。

## 技術要件

### ミドルウェア仕様
- **ファイル**: `src/middleware/auth.ts`
- **役割**: HTTPリクエストの認証処理
- **依存関係**: SessionService、Hono Context

### 主要機能

#### requireAuth
```typescript
export function requireAuth(): MiddlewareHandler<{ Bindings: Env }>
```
- セッションCookieの検証
- 認証が必要なエンドポイントでの必須認証チェック
- 未認証時の401レスポンス
- 認証済みユーザー情報のContext注入

#### optionalAuth
```typescript
export function optionalAuth(): MiddlewareHandler<{ Bindings: Env }>
```
- セッションCookieの検証（任意）
- 認証情報がある場合のContext注入
- 認証情報がない場合も処理継続

#### authContext
```typescript
export function getAuthContext(c: Context): AuthContext | null
```
- Contextから認証情報を取得
- 型安全な認証状態の確認
- ユーザー情報へのアクセス

### Context拡張

#### AuthContext
```typescript
interface AuthContext {
  user_id: string;
  session_id: string;
  auth_data: Auth0UserData;
  expires_at: string;
}
```

#### Context拡張
```typescript
interface AuthVariables {
  auth?: AuthContext;
}
```

### セッション処理

#### Cookie読み取り
- **Cookie名**: `session_id`
- **HTTPOnly**: true（セキュリティ考慮）
- **SameSite**: Lax
- **Secure**: プロダクションではtrue

#### セッション検証フロー
1. Cookieからsession_idを取得
2. SessionService.validateSessionで検証
3. 有効な場合、認証情報をContextに設定
4. 無効な場合、適切なエラーレスポンス

#### エラーハンドリング
- **401 Unauthorized**: セッション無効・未認証
- **500 Internal Server Error**: セッション検証時のシステムエラー
- **タイミング攻撃対策**: 定数時間でのエラーレスポンス

### セキュリティ考慮事項

#### セッション保護
- セッションID漏洩対策
- CSRF攻撃対策（SameSite Cookie）

#### エラーメッセージの統一
- セキュリティ情報の漏洩防止

- ログによる監査証跡

#### レート制限対応

### 使用例

#### 必須認証エンドポイント
```typescript
// src/api/v1/auth/me/get.ts
import { requireAuth } from '@/middleware/auth';

app.use(requireAuth());
app.get('/', async (c) => {
  const auth = getAuthContext(c);
  // auth is guaranteed to be non-null
  return c.json({ user_id: auth.user_id });
});
```

#### 任意認証エンドポイント
```typescript
// src/api/v1/public/get.ts
import { optionalAuth } from '@/middleware/auth';

app.use(optionalAuth());
app.get('/', async (c) => {
  const auth = getAuthContext(c);
  if (auth) {
    // 認証済みユーザー向けの処理
  } else {
    // ゲストユーザー向けの処理
  }
});
```
