## 概要

既存のGET /configエンドポイントを改修し、GET /api/v1/configsAPIを活用した完全な設定管理画面を実装する。現在の実装は基本的な設定表示・編集のみに対応しているが、設定項目の追加・削除・型変更・メタデータ編集に対応した包括的な管理UIに拡張する。

## 現在の実装状況

### 既存機能
- `src/config/get.ts`: 基本的な設定表示・編集画面
- `src/templates/config/form.tsx`: 設定フォームテンプレート
- パスワード認証による保護
- CSRF保護
- センシティブ情報のマスク表示

### 不足している機能
1. 設定項目の追加UI
2. 既存設定項目の削除UI
3. 設定項目の型変更機能
4. システム設定フラグの編集
5. 説明（description）の編集
6. バリデーションルールの設定

## 実装仕様

### 1. APIの活用方針

既存のRESTful Config APIを活用:
- `GET /api/v1/configs` - 設定一覧取得（ページネーション、フィルタリング対応）
- `POST /api/v1/configs` - 新規設定追加
- `PUT /api/v1/configs/:key` - 設定更新
- `DELETE /api/v1/configs/:key` - 設定削除

```javascript
// API連携機能
const ConfigAPI = {
  // 設定一覧取得
  async getConfigs(params = {}) {
    const query = new URLSearchParams(params);
    const response = await fetch(`/api/v1/configs?${query}`);
    return response.json();
  },
  
  // 設定追加
  async createConfig(configData) {
    const response = await fetch('/api/v1/configs', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(configData)
    });
    return response.json();
  },
  
  // 設定更新
  async updateConfig(key, configData) {
    const response = await fetch(`/api/v1/configs/${encodeURIComponent(key)}`, {
      method: 'PUT',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(configData)
    });
    return response.json();
  },
  
  // 設定削除
  async deleteConfig(key) {
    const response = await fetch(`/api/v1/configs/${encodeURIComponent(key)}`, {
      method: 'DELETE'
    });
    return response.json();
  }
};
```
