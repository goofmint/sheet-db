## 概要
設定管理システムの基盤APIとして、`GET /api/v1/configs`エンドポイントを実装する。このAPIは設定項目の一覧を取得する機能を提供し、フロントエンドの設定管理UIやダッシュボード画面で使用される。認証が必要なAPIとして実装し、設定値のマスキング機能や検索・フィルタリング機能を含む。

## 機能要件

### 設定項目一覧取得
- **目的**: 全設定項目の一覧を取得
- **機能**:
  - 全設定項目の一覧表示
  - システム設定とカスタム設定の区別
  - 機密情報のマスキング
  - ページネーション対応
  - 検索・フィルタリング機能

### 認証・セキュリティ
- **目的**: 設定情報の保護
- **機能**:
  - 管理者認証の必須化
  - 機密情報（パスワード、シークレット）のマスキング
  - アクセスログの記録

## 実装仕様

### APIハンドラー設計
```typescript
// src/api/v1/configs/index.ts
import { Context } from 'hono';
import { ConfigService } from '../../services/config';
import { checkConfigAuthentication } from '../../utils/auth';
import { maskSensitiveValue } from '../../utils/security';

export const getConfigsListHandler = async (c: Context<{ Bindings: Env }>) => {
  try {
    // 認証チェック
    const isAuthenticated = await checkConfigAuthentication(c);
    if (!isAuthenticated) {
      return c.json({
        success: false,
        error: {
          code: 'UNAUTHORIZED',
          message: 'Authentication required'
        }
      }, 401);
    }

    // クエリパラメータの取得・バリデーション
    const queryParams = extractAndValidateQueryParams(c);
    if (!queryParams.valid) {
      return c.json({
        success: false,
        error: {
          code: 'VALIDATION_ERROR',
          message: 'Invalid query parameters',
          details: queryParams.errors
        }
      }, 400);
    }

    // 設定項目の取得
    const result = await ConfigService.getConfigsList(queryParams.data);
    
    // 機密情報のマスキング
    const maskedConfigs = result.configs.map(config => ({
      ...config,
      value: maskSensitiveValue(config.key, config.value),
      validation: config.validation ? JSON.parse(config.validation) : null
    }));

    return c.json({
      success: true,
      data: {
        configs: maskedConfigs,
        pagination: result.pagination
      }
    });

  } catch (error) {
    console.error('Get configs list error:', error);
    return c.json({
      success: false,
      error: {
        code: 'INTERNAL_ERROR',
        message: 'Failed to retrieve configuration list'
      }
    }, 500);
  }
};

// クエリパラメータの抽出・バリデーション
function extractAndValidateQueryParams(c: Context) {
  const page = parseInt(c.req.query('page') || '1', 10);
  const limit = Math.min(parseInt(c.req.query('limit') || '50', 10), 100);
  const search = c.req.query('search') || '';
  const type = c.req.query('type') as 'string' | 'boolean' | 'number' | 'json' | undefined;
  const system = c.req.query('system') ? c.req.query('system') === 'true' : undefined;
  const sort = c.req.query('sort') as 'key' | 'type' | 'created_at' | 'updated_at' || 'key';
  const order = c.req.query('order') as 'asc' | 'desc' || 'asc';

  const errors: Record<string, string> = {};

  // バリデーション
  if (page < 1) {
    errors.page = 'Page must be greater than 0';
  }
  if (limit < 1) {
    errors.limit = 'Limit must be greater than 0';
  }
  if (type && !['string', 'boolean', 'number', 'json'].includes(type)) {
    errors.type = 'Type must be one of: string, boolean, number, json';
  }
  if (!['key', 'type', 'created_at', 'updated_at'].includes(sort)) {
    errors.sort = 'Sort must be one of: key, type, created_at, updated_at';
  }
  if (!['asc', 'desc'].includes(order)) {
    errors.order = 'Order must be asc or desc';
  }

  return {
    valid: Object.keys(errors).length === 0,
    errors,
    data: { page, limit, search, type, system, sort, order }
  };
}
```

### ConfigService拡張
```typescript
// src/services/config.ts に追加するメソッド

interface ConfigsListParams {
  page: number;
  limit: number;
  search: string;
  type?: 'string' | 'boolean' | 'number' | 'json';
  system?: boolean;
  sort: 'key' | 'type' | 'created_at' | 'updated_at';
  order: 'asc' | 'desc';
}

interface ConfigsListResult {
  configs: Config[];
  pagination: {
    total: number;
    page: number;
    limit: number;
    totalPages: number;
    hasNext: boolean;
    hasPrev: boolean;
  };
}

/**
 * 設定項目一覧を取得（ページネーション、検索、フィルタリング対応）
 */
static async getConfigsList(params: ConfigsListParams): Promise<ConfigsListResult> {
  const { page, limit, search, type, system, sort, order } = params;
  const offset = (page - 1) * limit;

  // クエリビルダーの構築
  let query = this.db.select().from(configTable);

  // 検索条件の追加
  const conditions = [];
  if (search) {
    conditions.push(
      or(
        like(configTable.key, `%${search}%`),
        like(configTable.description, `%${search}%`)
      )
    );
  }
  if (type) {
    conditions.push(eq(configTable.type, type));
  }
  if (system !== undefined) {
    conditions.push(eq(configTable.system_config, system ? 1 : 0));
  }

  if (conditions.length > 0) {
    query = query.where(and(...conditions));
  }

  // ソート条件の追加
  const sortColumn = {
    'key': configTable.key,
    'type': configTable.type,
    'created_at': configTable.created_at,
    'updated_at': configTable.updated_at
  }[sort];

  query = order === 'desc' 
    ? query.orderBy(desc(sortColumn))
    : query.orderBy(asc(sortColumn));

  // 総件数の取得
  const countQuery = this.db.select({ count: count() }).from(configTable);
  if (conditions.length > 0) {
    countQuery.where(and(...conditions));
  }
  const [{ count: total }] = await countQuery.execute();

  // ページネーション適用
  query = query.limit(limit).offset(offset);

  // データの取得
  const configs = await query.execute();

  // ページネーション情報の計算
  const totalPages = Math.ceil(total / limit);
  const hasNext = page < totalPages;
  const hasPrev = page > 1;

  return {
    configs,
    pagination: {
      total,
      page,
      limit,
      totalPages,
      hasNext,
      hasPrev
    }
  };
}
```

### セキュリティ機能

#### 機密情報マスキング
```typescript
// src/utils/security.ts
const SENSITIVE_KEYS = [
  'google.client_secret',
  'auth0.client_secret',
  'app.config_password',
  'storage.r2.secret_access_key'
];

const SENSITIVE_PATTERNS = [
  /password/i,
  /secret/i,
  /token/i,
  /key$/i
];

/**
 * 機密情報の値をマスキング
 */
export function maskSensitiveValue(configKey: string, value: string): string {
  // 明示的な機密キーのチェック
  if (SENSITIVE_KEYS.includes(configKey)) {
    return '*'.repeat(16);
  }

  // パターンマッチングによる機密情報の検出
  const isSensitive = SENSITIVE_PATTERNS.some(pattern => pattern.test(configKey));
  if (isSensitive) {
    return '*'.repeat Math.min(value.length, 16);
  }

  return value;
}
```

## エラーハンドリング

### エラーケース一覧
1. **認証エラー (401)**
   - 認証トークンが無効または期限切れ
   - 認証ヘッダーが不正

2. **権限エラー (403)**
   - 設定管理権限が不足

3. **バリデーションエラー (400)**
   - 無効なクエリパラメータ
   - ページ番号やリミットの範囲外指定

4. **システムエラー (500)**
   - データベース接続エラー
   - 予期しない内部エラー

### キャッシュ戦略
```typescript
// src/services/config.ts
const CONFIG_LIST_CACHE_KEY = 'config_list';
const CACHE_TTL = 5 * 60; // 5分

/**
 * キャッシュ機能付き設定一覧取得
 */
static async getConfigsListWithCache(params: ConfigsListParams): Promise<ConfigsListResult> {
  const cacheKey = `${CONFIG_LIST_CACHE_KEY}_${JSON.stringify(params)}`;
  
  // キャッシュからの取得を試行
  const cached = await this.cache.get(cacheKey);
  if (cached) {
    return JSON.parse(cached);
  }

  // データベースから取得
  const result = await this.getConfigsList(params);

  // キャッシュに保存
  await this.cache.set(cacheKey, JSON.stringify(result), { ttl: CACHE_TTL });

  return result;
}
```
