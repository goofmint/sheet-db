# 2.2.3 POST /api/v1/configs - 設定項目の追加

## ステータス
- [ ] 未完了
- 開始日: 2025-08-02
- 完了日: -

## 概要
設定項目を新規追加するAPIエンドポイントの実装。認証済みユーザーが新しい設定項目を追加できるようにする。

## 技術要件

### エンドポイント仕様
- **メソッド**: POST
- **パス**: `/api/v1/configs`
- **認証**: 必須（Authorizationヘッダーでconfig_passwordを使用）
- **Content-Type**: application/json

### リクエストボディ
```typescript
interface CreateConfigRequest {
  key: string;           // 設定キー（必須、ユニーク）
  value: string | number | boolean | object; // 設定値（必須）
  type: 'string' | 'number' | 'boolean' | 'json'; // 値の型（必須）
  description?: string;  // 説明（任意）
  system_config?: boolean; // システム設定フラグ（任意、デフォルト: false）
  validation?: {         // バリデーションルール（任意）
    min?: number;        // 最小値（number型の場合）
    max?: number;        // 最大値（number型の場合）
    pattern?: string;    // 正規表現パターン（string型の場合）
    enum?: any[];        // 許可値リスト
    required?: boolean;  // 必須フラグ
  };
}
```

### レスポンス

#### 成功時 (201 Created)
```json
{
  "success": true,
  "data": {
    "id": "d7f3c6e1-3e6b-46cf-a07c-e4d4ac9b7c1d",
    "key": "api_key",
    "value": "sk-abc123",
    "type": "string",
    "description": "API Key for external service",
    "system_config": false,
    "validation": {
      "pattern": "^sk-[a-zA-Z0-9]+$"
    },
    "created_at": "2025-08-02T10:00:00Z",
    "updated_at": "2025-08-02T10:00:00Z"
  }
}
```

#### エラーレスポンス

##### 400 Bad Request - バリデーションエラー
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid configuration data",
    "details": {
      "key": "Key is required",
      "type": "Invalid type specified"
    }
  }
}
```

##### 401 Unauthorized - 認証エラー
```json
{
  "success": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Authentication required"
  }
}
```

##### 409 Conflict - 重複エラー
```json
{
  "success": false,
  "error": {
    "code": "DUPLICATE_KEY",
    "message": "Configuration key already exists"
  }
}
```

##### 500 Internal Server Error
```json
{
  "success": false,
  "error": {
    "code": "INTERNAL_ERROR",
    "message": "Failed to create configuration"
  }
}
```

## 実装詳細

### バリデーション要件
1. **キーのバリデーション**
   - 必須項目
   - 1-255文字
   - 英数字、アンダースコア、ハイフン、ドットのみ許可
   - パターン: `/^[a-zA-Z0-9_.-]+$/`
   - 既存キーとの重複チェック

2. **値のバリデーション**
   - 必須項目
   - 型に応じた値の検証
   - JSON型の場合は有効なJSONであることを確認
   - 最大サイズ: 64KB

3. **型のバリデーション**
   - 必須項目
   - 許可値: `string`, `number`, `boolean`, `json`

4. **説明のバリデーション**
   - 任意項目
   - 最大1000文字

5. **バリデーションルールの検証**
   - 指定された型に対して適切なルールか確認
   - 正規表現パターンの妥当性チェック

### セキュリティ考慮事項
1. **入力サニタイゼーション**
   - SQLインジェクション対策（Drizzle ORMが自動的に処理）
   - XSS対策（Zodスキーマによる入力検証）

2. **アクセス制御**
   - Authorizationヘッダーでconfig_passwordによる認証が必須

### データベース操作
1. **トランザクション使用**
   - 設定作成時はトランザクション内で実行
   - エラー時は自動ロールバック

2. **タイムスタンプ**
   - `created_at`と`updated_at`を自動設定
