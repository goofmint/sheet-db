# [2.2.10] PUT /api/v1/configs/:keyを利用した/config での設定更新

## ステータス
- [ ] 未完了
- 開始日: 2025-08-05
- 完了日: 

## 概要
/config ページにおいて、PUT /api/v1/configs/:key APIを使用してクライアントサイドから既存の設定項目を更新できるようにする。

## 実装内容

### 1. インライン編集機能の実装
- **編集モードへの切り替え**
  - Editボタンクリックで該当行を編集モードに変更
  - 現在の値を入力フィールドに表示
  - 型に応じた適切な入力コントロールを表示

- **編集状態の管理**
  - 一度に編集できる行は1つのみ
  - ESCキーでキャンセル
  - 他の行のEditボタンクリック時は自動保存または確認

### 2. 値の更新処理
- **型別の値処理**
  - `string`: テキスト入力フィールド
  - `number`: 数値入力フィールド（バリデーション付き）
  - `boolean`: チェックボックス
  - `json`: テキストエリア（JSON構文検証付き）

- **バリデーション**
  - 必須チェック（空値の禁止）
  - 型別のフォーマット検証
  - 変更がない場合の更新スキップ

### 3. API呼び出しとエラーハンドリング
- **PUT /api/v1/configs/:key API呼び出し**
  - HttpOnly/SameSiteクッキーによるセッション認証
  - CSRFトークンのヘッダー送信（二重送信クッキーパターン）
  - リクエストボディに新しい値を含める
  - レスポンス処理（成功/エラー）

- **エラー処理**
  - APIエラーの適切な表示
  - 楽観的UIの実装（更新中の状態表示）
  - 失敗時の値のロールバック

### 4. UIフィードバック
- **更新中の表示**
  - ローディングインジケーター
  - 編集フィールドの無効化
  - 保存ボタンの状態変更

- **成功時の処理**
  - 編集モードから表示モードへの切り替え
  - 成功メッセージの表示（一時的）
  - 更新された値の即時反映

- **失敗時の処理**
  - エラーメッセージの表示
  - 編集モードの維持
  - 元の値へのロールバックオプション

### 5. キーボードショートカット
- **Enter**: 変更を保存
- **ESC**: 編集をキャンセル
- **Tab**: 次のフィールドへ移動（複数フィールドの場合）

- XSS対策（編集値のサニタイゼーション）
- 同時編集の制御（楽観的ロック）

- 明確な編集状態の表示
- キーボードナビゲーション対応
- モバイルデバイスでの操作性

### パフォーマンス

- デバウンス処理（連続的な更新の防止）
- 楽観的UI更新による体感速度の向上
