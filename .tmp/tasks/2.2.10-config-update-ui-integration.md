# [2.2.10] PUT /api/v1/configs/:keyを利用した/config での設定更新

## ステータス
- [ ] 未完了
- 開始日: 2025-08-05
- 完了日: 

## 概要
/config ページにおいて、PUT /api/v1/configs/:key APIを使用してクライアントサイドから既存の設定項目を更新できるようにする。

## 実装内容

### 1. モーダル編集機能の実装
- **編集モーダルの表示**
  - Editボタンクリックで既存のモーダルを編集モードで表示
  - 現在の値をフォームフィールドにプリセット
  - キーフィールドは読み取り専用で表示
  - 型に応じた適切な入力コントロールを表示

- **モーダルの再利用**
  - 既存の「Add Configuration」モーダルを編集用に活用
  - モーダルタイトルを「Edit Configuration」に動的変更
  - 送信ボタンのテキストを「Update」に変更
  - フォームのアクション（追加/更新）を状態で管理

### 2. 値の更新処理
- **型別の値処理**
  - `string`: テキスト入力フィールド
  - `number`: 数値入力フィールド（バリデーション付き）
  - `boolean`: チェックボックス
  - `json`: テキストエリア（JSON構文検証付き）

- **バリデーション**
  - 必須チェック（空値の禁止）
  - 型別のフォーマット検証
  - 変更がない場合の更新スキップ

### 3. API呼び出しとエラーハンドリング
- **PUT /api/v1/configs/:key API呼び出し**
  - Bearer認証ヘッダーによる認証（config_password使用）
  - リクエストボディに新しい値を含める
  - レスポンス処理（成功/エラー）

- **エラー処理**
  - APIエラーの適切な表示
  - 楽観的UIの実装（更新中の状態表示）
  - 失敗時の値のロールバック

### 4. UIフィードバック
- **更新中の表示**
  - モーダル内でのローディングインジケーター
  - フォームフィールドの無効化
  - Updateボタンの状態変更

- **成功時の処理**
  - モーダルの自動クローズ
  - 成功メッセージの表示（一時的）
  - テーブル内の更新された値の即時反映

- **失敗時の処理**
  - モーダル内でのエラーメッセージ表示
  - モーダルは開いたまま維持
  - 修正して再試行可能

### 5. モーダル状態管理
- **追加モードと編集モードの切り替え**
  - グローバル状態で現在のモード（add/edit）を管理
  - 編集時は対象の設定キーと現在値を保持
  - モーダルクローズ時に状態をリセット

- **フォーム初期化**
  - 追加モード：全フィールドを空にリセット
  - 編集モード：既存値でフィールドを初期化
  - キーフィールドの編集可否を動的制御
