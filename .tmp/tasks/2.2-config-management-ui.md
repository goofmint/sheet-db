## 概要
Configテーブルに保存された設定データを動的に取得・表示し、管理者が柔軟に設定を変更できるWebベースの設定管理UIを実装する。設定画面は `/config` でアクセス可能。

## 機能要件

### 設定管理画面の表示

- **エンドポイント**: `GET /config`
- **目的**: パスワード認証後、設定データを一覧表示する管理画面を提供
- **機能**:
  - パスワード入力フォーム（未認証時）
  - 設定一覧表（認証後）
  - 各設定項目の編集フォーム


## 実装仕様

### GET /config - 設定管理画面

#### 処理フロー
1. パスワード認証確認（Cookieまたはセッション）
2. 未認証の場合：パスワード入力フォームを表示
3. 認証済みの場合：
   - ConfigServiceから全設定を取得
   - HTMLとして設定一覧画面を返却

#### HTML構成
```html
<!-- 未認証時 -->
<form method="post" action="/config/auth">
  <input type="password" name="password" placeholder="設定パスワード">
  <button type="submit">ログイン</button>
</form>

<!-- 認証済み時 -->
<table>
  <tr>
    <th>設定キー</th>
    <th>値</th>
    <th>説明</th>
  </tr>
  <tr>
    <td>google.clientId</td>
    <td><input type="text" value="123..."></td>
    <td>Google OAuth2 クライアントID</td>
  </tr>
  <!-- 他の設定項目 -->
</table>
```

## UI設計

### シンプルな設定一覧画面
- パスワード認証フォーム（未認証時）
- 設定一覧テーブル（認証済み時）
  - キー
  - 値（編集可能なinputフィールド）
  - 説明（読み取り専用）
- 保存ボタン（次のタスクで実装）

## 実装内容

### 作成ファイル
```
src/api/v1/config/
└── get.ts  # GET /config 設定管理画面
```

### 実装詳細

#### GET /config エンドポイント
1. パスワード認証の確認（Cookie: config_auth）
2. 未認証時：パスワード入力フォームのHTML返却
3. 認証済み時：
   - ConfigService.getAll()で全設定取得
   - 設定一覧テーブルのHTML生成
   - 機密情報（パスワード等）はマスキング表示

#### セキュリティ
- パスワードはCookieに保存（httpOnly, secure）
- 機密データは `****` でマスキング表示

## テスト要件

### GET /config のテスト
1. **未認証時**
   - パスワード入力フォームが表示される
   - HTMLレスポンスが返される

2. **認証済み時**
   - 設定一覧テーブルが表示される
   - 機密データがマスキングされている
   - 全設定項目が表示される

3. **エラーハンドリング**
   - ConfigService.getAll()のエラー時の対応
