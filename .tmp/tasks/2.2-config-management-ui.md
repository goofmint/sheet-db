## 概要
Configテーブルに保存された設定データを動的に取得・表示し、管理者が柔軟に設定を変更できるWebベースの設定管理UIを実装する。設定画面は `/config` でアクセス可能。

## 機能要件

### 2.2.1 設定データ取得・表示機能

- 最初にパスワード認証を行い、設定管理画面にアクセス

- **エンドポイント**: `GET /api/v1/setup` (既存APIの拡張)
- **目的**: 現在の設定データを動的に取得し、フォームで編集可能な形式で表示
- **特徴**: 新しい設定項目の追加に対応できる柔軟な設計、既存APIとの互換性維持

### 2.2.2 設定データ更新機能  
- **エンドポイント**: `POST /api/v1/setup` (既存APIの拡張)
- **目的**: フォームからの設定変更をConfigテーブルに反映
- **特徴**: 設定項目ごとの型検証とトランザクション処理、部分更新対応

## UI設計

### ページ構成
```
GET /config - 設定管理画面
├── 認証セクション（パスワード入力）
├── 設定グループ別タブ
│   ├── Google設定
│   ├── Auth0設定  
│   ├── アプリケーション設定
│   └── ストレージ設定
├── 保存・リセットボタン
└── 変更履歴表示
```

### レスポンシブデザイン要件
- **デスクトップ**: 3カラムレイアウト（ナビ・フォーム・プレビュー）
- **タブレット**: 2カラムレイアウト（フォーム・プレビュー）
- **モバイル**: 1カラムレイアウト（スタック形式）

### フォーム要素の動的生成

#### 設定項目の型別表示
```typescript
interface ConfigField {
  key: string;
  value: any;
  type: 'string' | 'number' | 'boolean' | 'json';
  description: string;
  required: boolean;
  sensitive: boolean;
  options?: string[]; // select box用
  validation?: {
    pattern?: string;
    min?: number;
    max?: number;
    minLength?: number;
    maxLength?: number;
  };
}
```

#### フィールドタイプ別UI
- **string**: テキスト入力（sensitive=trueの場合はパスワード型）
- **number**: 数値入力（min/max制限付き）
- **boolean**: チェックボックス
- **json**: テキストエリア（JSON validation付き）
- **options**: セレクトボックス

1. **既存APIとの互換性維持**

   - 認証状態によってレスポンスを切り替え
   - 既存のセットアップフローを壊さない

2. **動的フォーム生成**

   - 既存の設定項目の表示と編集
   - 設定グループごとのタブ表示

3. **セキュリティ考慮**
   - 機密データ（パスワード、シークレット）のマスキング
   - 設定変更時の再認証フロー
   - ログインセッションとの連携

4. **ユーザビリティ**
   - 直感的なフォーム設計
   - リアルタイムバリデーション
   - 変更内容の視覚的な確認

   - 設定データのキャッシュ戦略
   - 部分更新での効率化

### ConfigService拡張案

既存のConfigServiceクラスに以下のメソッドを追加：

```typescript
// src/services/config.ts に追加するメソッド
class ConfigService {
  // 既存のメソッド...
  
  // 設定グループ管理
  static getConfigGroups(): ConfigGroup[];
  static getConfigsByGroup(group: string): ConfigField[];
  static getAllConfigsWithMetadata(): Promise<Record<string, ConfigField>>;
  
  // バリデーション連携
  static validateConfig(key: string, value: any): ValidationResult;
  static validateAllConfigs(configs: Record<string, any>): ValidationResult[];
  
  // 部分更新サポート
  static updateConfigs(updates: Record<string, any>): Promise<UpdateResult>;
  
  // セキュリティ機能
  static maskSensitiveValues(configs: Record<string, any>): Record<string, any>;
  static isSensitiveKey(key: string): boolean;
}
```

### 設定項目の分類

#### Google設定
- `google.clientId` - OAuth2クライアントID
- `google.clientSecret` - OAuth2クライアントシークレット  
- `google.sheetId` - メインスプレッドシートID

#### Auth0設定
- `auth0.domain` - Auth0ドメイン
- `auth0.clientId` - Auth0クライアントID
- `auth0.clientSecret` - Auth0クライアントシークレット
- `auth0.audience` - APIオーディエンス（オプション）
- `auth0.scope` - OAuth2スコープ

#### アプリケーション設定
- `app.configPassword` - 設定画面パスワード
- `app.setupCompleted` - セットアップ完了フラグ

#### ストレージ設定
- `storage.type` - ストレージタイプ（r2 | google_drive）

### エラーハンドリング戦略

1. **バリデーションエラー**
   - フィールドレベルでの詳細エラー
   - 日本語エラーメッセージ
   - 修正案の提示

2. **システムエラー**
   - データベース接続エラー

3. **ユーザーエラー**
