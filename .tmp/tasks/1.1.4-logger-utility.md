# タスク 1.1.4: ログユーティリティの設計・実装

## 概要
Cloudflare Workers環境で動作する統一されたログシステムを構築します。

## 要件

### 機能要件
1. **ログレベル制御**
   - `debug`, `info`, `warn`, `error`の4段階
   - 環境変数`LOG_LEVEL`による動的制御
   - デフォルトは`info`レベル

2. **構造化ログ**
   - JSON形式での出力
   - タイムスタンプ、ログレベル、メッセージの必須項目
   - 追加メタデータの柔軟な付与

3. **コンテキスト情報**
   - リクエストID（追跡用）
   - ユーザーID（認証後）
   - APIパス・メソッド
   - 実行時間測定

4. **エラー詳細**
   - エラースタックトレース
   - エラーコード・カテゴリ
   - 関連するリクエスト情報

### 非機能要件
1. **パフォーマンス**
   - 低オーバーヘッド
   - 非同期処理でのブロッキング回避

2. **Cloudflare Workers対応**
   - console API使用
   - Workers固有の制約対応

3. **開発効率**
   - 簡潔なAPI
   - TypeScript完全対応

## 設計

### インターフェース
```typescript
// ログレベル
type LogLevel = 'debug' | 'info' | 'warn' | 'error';

// ログエントリ
interface LogEntry {
  timestamp: string;
  level: LogLevel;
  message: string;
  metadata?: Record<string, unknown>;
  requestId?: string;
  userId?: string;
  path?: string;
  method?: string;
  duration?: number;
  error?: {
    name: string;
    message: string;
    stack?: string;
    code?: string;
  };
}

// ロガークラス
class Logger {
  constructor(level: LogLevel = 'info', context?: Record<string, unknown>);
  
  debug(message: string, metadata?: Record<string, unknown>): void;
  info(message: string, metadata?: Record<string, unknown>): void;
  warn(message: string, metadata?: Record<string, unknown>): void;
  error(message: string, error?: Error, metadata?: Record<string, unknown>): void;
  
  // コンテキスト付きロガー作成
  withContext(context: Record<string, unknown>): Logger;
  
  // 実行時間測定
  time(label: string): () => void;
}

// ファクトリー関数
function createLogger(env: Env, context?: Record<string, unknown>): Logger;
```

### コンテキスト管理
```typescript
// リクエストコンテキスト
interface RequestContext {

// Honoミドルウェアでコンテキスト設定
function loggingMiddleware(): MiddlewareHandler;
```

```typescript
// ログ設定
interface LogConfig {
  level: LogLevel;
  enableConsole: boolean;

  includeStackTrace: boolean;

// 環境変数から設定を取得
```

## 使用例

### 基本的な使用
```typescript
import { createLogger } from '../utils/logger';

const logger = createLogger(env);

logger.info('Application started');
logger.debug('Processing request', { path: '/api/data' });
logger.error('Database connection failed', error, { table: 'users' });
```

### ミドルウェアでのコンテキスト設定
```typescript
import { loggingMiddleware } from '../middleware/logging';

app.use(loggingMiddleware());

// APIハンドラー内で自動的にコンテキスト付きロガーが利用可能
app.get('/api/data', async (c) => {
  const logger = c.get('logger'); // コンテキスト付きロガー
  logger.info('Data request received');
});
```

### 実行時間測定
```typescript
const timer = logger.time('database_query');
const result = await db.select().from(table);
timer(); // 自動的に実行時間をログ出力
```
