# タスク 1.1.4: ログユーティリティの設計・実装

## 概要
現段階では、Honoの標準ロガーミドルウェアを使用してログ機能を実装します。将来的にはより高機能なログシステムへの拡張を想定。

## 現在の実装方針（Phase 1）

### 基本方針
- **Honoの標準ロガーミドルウェア**を使用
- シンプルで軽量な実装
- 必要最小限の設定で開始

### 機能要件
1. **基本ログ出力**
   - HTTPリクエスト・レスポンスの自動ログ
   - メソッド、パス、ステータスコード、実行時間
   - console.logへの出力

2. **設定可能性**
   - 環境変数によるログ有効/無効の制御
   - カスタムフォーマット対応


## 現在の実装

### Honoロガーミドルウェアの使用
```typescript
import { Hono } from 'hono';
import { logger } from 'hono/logger';

const app = new Hono<{ Bindings: Env }>();

// ログミドルウェアの適用
app.use(logger());

// または条件付きで適用
app.use(logger((message, ...rest) => {
  // 環境変数でログ制御
  if (env.LOG_LEVEL !== 'none') {
    console.log(message, ...rest);
  }
}));
```

### 設定例
```typescript
// 環境変数による制御
interface Env {
  LOG_LEVEL?: 'debug' | 'info' | 'warn' | 'error' | 'none';
  // その他の環境変数...
}

// カスタムフォーマット例
app.use('*', logger((message) => {
  const timestamp = new Date().toISOString();
  console.log(`[${timestamp}] ${message}`);
}));
```

## 使用例

### 現在の実装（Honoロガー）
```typescript
// src/index.ts
import { Hono } from 'hono';
import { logger } from 'hono/logger';

const app = new Hono<{ Bindings: Env }>();

// 基本的なログミドルウェア適用
app.use(logger());

export default app;
```

### 条件付きログ出力例
```typescript
// src/index.ts
import { Hono } from 'hono';
import { logger } from 'hono/logger';

const app = new Hono<{ Bindings: Env }>();

// 環境変数に応じたログ制御
app.use((c, next) => {
  if (c.env.LOG_LEVEL !== 'none') {
    return logger()(c, next);
  }
  return next();
});

export default app;
```

### APIハンドラー内での追加ログ
```typescript
// 各APIハンドラー内で
app.get('/api/data', async (c) => {
  console.log('Processing data request', { 
    path: c.req.path,
    method: c.req.method 
  });
  
  try {
    const result = await someOperation();
    return c.json(result);
  } catch (error) {
    console.error('Operation failed:', error);
    return c.json({ error: 'Internal error' }, 500);
  }
});
```
