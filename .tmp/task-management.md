# タスク管理システム - sheetDBプロジェクト

## 概要

このドキュメントでは、sheetDBプロジェクトにおけるタスク管理手法を定義します。

## フォルダ構造

```
.tmp/
├── tasks.md                   # メインタスクリスト（全体進捗管理）
├── tasks/                     # 個別タスク詳細
│   ├── 1.1.1-project-setup.md
│   ├── 1.1.2-types-definition.md
│   ├── 1.2-database-schema.md
│   └── ...
├── requirements.md            # 要件定義
├── design.md                  # 技術設計
└── [その他設計ドキュメント]
```

## タスクのナンバリング規則

### フェーズ番号体系
- **Phase 1**: 基盤構築 (1.x.x)
- **Phase 2**: 認証・認可 (2.x.x)
- **Phase 3**: Google Sheets統合 (3.x.x)
- **Phase 4**: キャッシュ・最適化 (4.x.x)
- **Phase 5**: 拡張機能 (5.x.x)

### サブタスクの分類
- **1.1.x**: プロジェクト構造のセットアップ
- **1.2.x**: D1データベーススキーマの実装
- **1.3.x**: 基本的なAPIルーティング

### ファイル命名規則
```
[フェーズ].[カテゴリ].[タスク番号]-[タスク名].md
```

例:
- `1.1.1-project-setup.md` → Phase 1, カテゴリ1, タスク1
- `1.2-database-schema.md` → Phase 1, カテゴリ2（大きなタスク）
- `2.1.3-jwt-verification.md` → Phase 2, カテゴリ1, タスク3

## タスクステータス管理

### メインタスクリスト (tasks.md)
- `[ ]` 未着手
- `[x]` 完了

### 個別タスクファイル
各タスクファイルには以下の情報を含める：

```markdown
# [タスク番号] タスク名

## ステータス
- [x] 完了 / [ ] 未完了
- 開始日: YYYY-MM-DD
- 完了日: YYYY-MM-DD

## 概要
タスクの目的と期待される成果

## 実装内容
具体的な実装項目のチェックリスト

## 関連ファイル
- 作成/変更されるファイル一覧

## 依存関係
- 前提となるタスク
- このタスクに依存するタスク

## メモ
実装時の注意点や学習内容
```

## ワークフロー

### 1. タスク開始時
1. 該当タスクを`tasks.md`で確認
2. `.tmp/tasks/`から対応する詳細ファイルを開く
3. 実装開始前に依存関係を確認
4. TodoWriteツールでタスクを`in_progress`に設定

### 2. タスク実行中
1. 実装内容のチェックリストを順次完了
2. 関連ファイルの作成/変更を記録
3. 問題や発見事項をメモセクションに記録

### 3. タスク完了時
1. すべてのチェック項目が完了していることを確認
2. テストとリントの実行
3. TodoWriteツールでタスクを`completed`に設定
4. `tasks.md`の対応項目を`[x]`に変更
5. 次のタスクの準備

### 4. フェーズ完了時
1. 全タスクの完了確認
2. 統合テストの実行
3. ドキュメントの最終更新
4. 次フェーズの準備

## Git管理

### ブランチ戦略
- 新機能開発: `feature/task-[番号]-[概要]`
- バグ修正: `fix/issue-[概要]`
- ドキュメント: `docs/[概要]`

### コミット規則
```
[タスク番号] 概要: 具体的な変更内容

詳細説明（必要に応じて）
```

例:
```
1.2.1 feat: Configテーブルのスキーマ定義を追加

- Drizzle ORMでのスキーマ定義
- 型安全性を確保したインターフェース
- バリデーション機能の実装
```
